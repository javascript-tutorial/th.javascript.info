# Rest parameters และ spread syntax

ฟังก์ชัน built-in หลายตัวของ JavaScript รองรับอาร์กิวเมนต์จำนวนไม่จำกัด

ยกตัวอย่างเช่น:

- `Math.max(arg1, arg2, ..., argN)` -- คืนค่าจำนวนที่มากที่สุดจากอาร์กิวเมนต์ทั้งหมด
- `Object.assign(dest, src1, ..., srcN)` -- คัดลอกพร็อพเพอร์ตี้จาก `src1..N` ไปยัง `dest`
- ...และอื่น ๆ อีกมากมาย

ในบทนี้ เราจะมาเรียนรู้วิธีการแบบเดียวกัน รวมถึงวิธีส่งอาร์เรย์เข้าไปเป็นพารามิเตอร์ของฟังก์ชัน

## Rest parameters `...`

เราเรียกฟังก์ชันโดยส่งอาร์กิวเมนต์กี่ตัวก็ได้ ไม่ว่าฟังก์ชันนั้นจะประกาศพารามิเตอร์ไว้กี่ตัวก็ตาม

ลองดูตัวอย่าง:
```js run
function sum(a, b) {
  return a + b;
}

alert( sum(1, 2, 3, 4, 5) );
```

โค้ดนี้จะไม่เกิด error แม้จะส่งอาร์กิวเมนต์มากเกินไป แต่ผลลัพธ์จะคำนวณจากอาร์กิวเมนต์ 2 ตัวแรกเท่านั้น

เราสามารถรวมพารามิเตอร์ที่เหลือเข้าด้วยกันได้ โดยใช้จุดสามจุด `...` ตามด้วยชื่อของอาร์เรย์ที่จะรองรับค่าเหล่านั้น จุดสามจุดนี้หมายความว่า "รวมพารามิเตอร์ที่เหลือทั้งหมดเข้าไปในอาร์เรย์"

ยกตัวอย่าง การรวมอาร์กิวเมนต์ทั้งหมดเข้าในอาร์เรย์ `args`:

```js run
function sumAll(...args) { // args คือชื่อของ array
  let sum = 0;

  for (let arg of args) sum += arg;

  return sum;
}

alert( sumAll(1) ); // 1
alert( sumAll(1, 2) ); // 3
alert( sumAll(1, 2, 3) ); // 6
```

เราเลือกให้พารามิเตอร์ตัวแรก ๆ เป็นตัวแปรแยก แล้วรวมเฉพาะส่วนที่เหลือไว้ใน rest parameters ก็ได้

ตัวอย่างนี้ให้ 2 ตัวแรกเป็นตัวแปร แล้วที่เหลือรวมไว้ในอาร์เรย์ `titles`

```js run
function showName(firstName, lastName, ...titles) {
  alert( firstName + ' ' + lastName ); // Julius Caesar

  // พารามิเตอร์ที่เหลือจะรวมอยู่ในอาร์เรย์ titles
  // เช่น titles = ["Consul", "Imperator"]
  alert( titles[0] ); // Consul
  alert( titles[1] ); // Imperator
  alert( titles.length ); // 2
}

showName("Julius", "Caesar", "Consul", "Imperator");
```

````warn header="Rest parameters ต้องอยู่ท้ายสุดเสมอ"
เนื่องจาก rest parameters จะรวมอาร์กิวเมนต์ที่เหลือทั้งหมด การเขียนแบบนี้จึงไม่สมเหตุสมผลและทำให้เกิด error:

```js
function f(arg1, ...rest, arg2) { // arg2 หลังจาก ...rest ?!
  // error
}
```

ฉะนั้น `...rest` ต้องอยู่ท้ายสุด
````

## ตัวแปร "arguments"

นอกจากนี้ยังมีออบเจ็กต์พิเศษที่มีลักษณะคล้ายอาร์เรย์ ชื่อว่า `arguments` ซึ่งเก็บอาร์กิวเมนต์ทั้งหมดเรียงตาม index

ลองดูตัวอย่าง

```js run
function showName() {
  alert( arguments.length );
  alert( arguments[0] );
  alert( arguments[1] );

  // วนลูปได้เช่นกัน
  // for(let arg of arguments) alert(arg);
}

// แสดงผลเป็น: 2, Julius, Caesar
showName("Julius", "Caesar");

// แสดงผลเป็น: 1, Ilya, undefined (ไม่มีอาร์กิวเมนต์ตัวที่สอง)
showName("Ilya");
```

สมัยก่อนที่จะมี rest parameters การใช้ `arguments` เป็นวิธีเดียวที่จะเข้าถึงอาร์กิวเมนต์ทั้งหมดของฟังก์ชัน ซึ่งยังใช้งานได้อยู่ และอาจเจอได้ในโค้ดเก่า ๆ

แต่ข้อเสียของ `arguments` คือ ถึงจะมีลักษณะคล้ายอาร์เรย์และวนลูปได้ แต่จริง ๆ แล้วไม่ใช่อาร์เรย์ จึงใช้เมธอดของอาร์เรย์ไม่ได้ เช่น `arguments.map(...)` จะใช้ไม่ได้

นอกจากนี้ `arguments` จะเก็บอาร์กิวเมนต์ทั้งหมดเสมอ เราไม่สามารถเลือกรับเฉพาะบางส่วนได้แบบที่ rest parameters ทำได้

ดังนั้นถ้าต้องการความยืดหยุ่น rest parameters จึงเหมาะกว่า

````smart header="Arrow function ไม่มี `\"arguments\"`"
ถ้าเราเข้าถึงออบเจ็กต์ `arguments` จากภายใน arrow function จะได้ค่า `arguments` ของฟังก์ชันภายนอกที่ครอบอยู่แทน

ตัวอย่าง:

```js run
function f() {
  let showArg = () => alert(arguments[0]);
  showArg();
}

f(1); // 1
```

ดังที่เราเคยเรียนมาแล้ว arrow function ไม่มี `this` เป็นของตัวเอง และตอนนี้เราก็รู้แล้วว่าไม่มีออบเจ็กต์ `arguments` เป็นของตัวเองเช่นกัน
````


## Spread syntax [#spread-syntax]

เราเพิ่งเห็นวิธีรวมกลุ่มพารามิเตอร์เข้าเป็นอาร์เรย์ไปแล้ว

แต่บางครั้งเราต้องการทำสิ่งตรงกันข้าม

ยกตัวอย่าง ฟังก์ชัน built-in อย่าง [Math.max](mdn:js/Math/max) จะคืนค่าจำนวนที่มากที่สุดจากรายการที่ส่งเข้าไป:

```js run
alert( Math.max(3, 5, 1) ); // 5
```

สมมติเรามีอาร์เรย์ `[3, 5, 1]` แล้วจะส่งเข้า `Math.max` ได้อย่างไร?

ส่งเข้าไปตรง ๆ ไม่ได้ เพราะ `Math.max` รับอาร์กิวเมนต์เป็นตัวเลขแยกทีละตัว ไม่ใช่อาร์เรย์:

```js run
let arr = [3, 5, 1];

*!*
alert( Math.max(arr) ); // NaN
*/!*
```

แน่นอนว่าเราเขียน `Math.max(arr[0], arr[1], arr[2])` แบบระบุทีละตัวไม่ได้ เพราะไม่รู้ว่าอาร์เรย์จะมีสมาชิกกี่ตัว อาจมีเยอะมากหรืออาจไม่มีเลยก็ได้

*Spread syntax* จะเข้ามาช่วยตรงนี้ หน้าตาเหมือน rest parameters เพราะใช้ `...` เหมือนกัน แต่ทำงานตรงกันข้าม

เมื่อใช้ `...arr` ตอนเรียกฟังก์ชัน จะ "กระจาย" iterable ออบเจ็กต์ `arr` ออกเป็นอาร์กิวเมนต์แยกทีละตัว

ลองดูตัวอย่างกับ `Math.max`:

```js run
let arr = [3, 5, 1];

alert( Math.max(...arr) ); // 5 (spread กระจายอาร์เรย์ออกเป็นอาร์กิวเมนต์แยกทีละตัว)
```

นอกจากนี้ยังส่ง iterable หลายตัวพร้อมกันได้:

```js run
let arr1 = [1, -2, 3, 4];
let arr2 = [8, 3, -8, 1];

alert( Math.max(...arr1, ...arr2) ); // 8
```

หรือจะผสม spread syntax กับค่าตัวเลขธรรมดาก็ได้:


```js run
let arr1 = [1, -2, 3, 4];
let arr2 = [8, 3, -8, 1];

alert( Math.max(1, ...arr1, 2, ...arr2, 25) ); // 25
```

Spread syntax ยังใช้รวมอาร์เรย์เข้าด้วยกันได้อีกด้วย:

```js run
let arr = [3, 5, 1];
let arr2 = [8, 9, 15];

*!*
let merged = [0, ...arr, 2, ...arr2];
*/!*

alert(merged); // 0,3,5,1,2,8,9,15 (0, ต่อด้วย arr, ต่อด้วย 2, ต่อด้วย arr2)
```

ตัวอย่างข้างต้นใช้อาร์เรย์ แต่จริง ๆ แล้ว spread syntax ใช้ได้กับ iterable ทุกชนิด

ยกตัวอย่างเช่น ใช้ spread syntax แปลงสตริงเป็นอาร์เรย์ของตัวอักษร:

```js run
let str = "Hello";

alert( [...str] ); // H,e,l,l,o
```

Spread syntax ทำงานเบื้องหลังโดยใช้ iterator เหมือนกับ `for..of`

สำหรับสตริง `for..of` จะคืนค่าเป็นตัวอักษรทีละตัว ดังนั้น `...str` จึงได้ `"H","e","l","l","o"` แล้วพอส่งเข้า `[...str]` ก็จะได้อาร์เรย์ของตัวอักษรออกมา

งานแบบนี้เราใช้ `Array.from` ก็ได้เช่นกัน เพราะมันแปลง iterable (เช่น สตริง) ให้เป็นอาร์เรย์ได้:

```js run
let str = "Hello";

// Array.from แปลง iterable ให้เป็นอาร์เรย์
alert( Array.from(str) ); // H,e,l,l,o
```

ผลลัพธ์ที่ได้เหมือนกับ `[...str]`

แต่ `Array.from(obj)` กับ `[...obj]` มีความต่างเล็กน้อย:

- `Array.from` ใช้ได้ทั้งกับ array-like และ iterable
- Spread syntax ใช้ได้กับ iterable เท่านั้น

ดังนั้นถ้าต้องการแปลงอะไรสักอย่างให้เป็นอาร์เรย์ `Array.from` จึงครอบคลุมกว่า


## การคัดลอก array/object

จำได้ไหมว่าเราเคยพูดถึง `Object.assign()` [ไปก่อนหน้านี้](info:object-copy#cloning-and-merging-object-assign)?

เราใช้ spread syntax ทำแบบเดียวกันได้เลย

```js run
let arr = [1, 2, 3];

*!*
let arrCopy = [...arr]; // กระจายอาร์เรย์ออกเป็นรายการค่า
                        // แล้วนำไปสร้างอาร์เรย์ตัวใหม่
*/!*

// อาร์เรย์ทั้งสองมีค่าเหมือนกันไหม?
alert(JSON.stringify(arr) === JSON.stringify(arrCopy)); // true

// แต่อ้างอิงตัวเดียวกันไหม?
alert(arr === arrCopy); // false (คนละตัวกัน)

// แก้ไขต้นฉบับจะไม่กระทบสำเนา:
arr.push(4);
alert(arr); // 1, 2, 3, 4
alert(arrCopy); // 1, 2, 3
```

ใช้วิธีเดียวกันนี้คัดลอกออบเจ็กต์ก็ได้เช่นกัน:

```js run
let obj = { a: 1, b: 2, c: 3 };

*!*
let objCopy = { ...obj }; // กระจายออบเจ็กต์ออกเป็นพร็อพเพอร์ตี้แยกทีละตัว
                          // แล้วนำไปสร้างออบเจ็กต์ตัวใหม่
*/!*

// ออบเจ็กต์ทั้งสองมีค่าเหมือนกันไหม?
alert(JSON.stringify(obj) === JSON.stringify(objCopy)); // true

// แต่อ้างอิงตัวเดียวกันไหม?
alert(obj === objCopy); // false (คนละตัวกัน)

// แก้ไขต้นฉบับจะไม่กระทบสำเนา:
obj.d = 4;
alert(JSON.stringify(obj)); // {"a":1,"b":2,"c":3,"d":4}
alert(JSON.stringify(objCopy)); // {"a":1,"b":2,"c":3}
```

วิธีนี้เขียนสั้นกว่า `let objCopy = Object.assign({}, obj)` หรือ `let arrCopy = Object.assign([], arr)` มาก จึงนิยมใช้กันมากกว่า


## สรุป

พอเห็น `"..."` ในโค้ด อาจเป็นได้ทั้ง rest parameters หรือ spread syntax

แยกได้ง่าย ๆ ดังนี้:

- ถ้า `...` อยู่ท้ายพารามิเตอร์ของฟังก์ชัน นั่นคือ "rest parameters" ซึ่งจะรวมอาร์กิวเมนต์ที่เหลือเข้าเป็นอาร์เรย์
- ถ้า `...` อยู่ตอนเรียกฟังก์ชัน นั่นคือ "spread syntax" ซึ่งจะกระจายอาร์เรย์ออกเป็นอาร์กิวเมนต์แยกทีละตัว

รูปแบบการใช้งาน:

- Rest parameters ใช้สร้างฟังก์ชันที่รับอาร์กิวเมนต์ได้ไม่จำกัดจำนวน
- Spread syntax ใช้ส่งอาร์เรย์เข้าไปในฟังก์ชันที่ต้องการอาร์กิวเมนต์แยกทีละตัว

ทั้งสองอย่างนี้ช่วยให้เราสลับไปมาระหว่างรายการค่ากับอาร์เรย์ได้สะดวก

นอกจากนี้ ยังสามารถเข้าถึงอาร์กิวเมนต์ทั้งหมดของฟังก์ชันได้ผ่าน `arguments` แบบเดิม ซึ่งเป็น array-like iterable ออบเจ็กต์
