# Constructor และตัวดำเนินการ "new"

การใช้ `{...}` ช่วยให้เราสร้างออบเจ็กต์ได้ แต่ในกรณีที่ต้องการสร้างออบเจ็กต์หลายๆ ตัวที่มีลักษณะคล้ายกัน เช่น ผู้ใช้หลายคน, เมนูหลายรายการ ฯลฯ การใช้ `{...}` ซ้ำๆ อาจไม่ใช่วิธีที่ดี 

ตรงนี้เราใช้ฟังก์ชัน constructor ร่วมกับตัวดำเนินการ `"new"` ได้

## ฟังก์ชัน Constructor

จริงๆ แล้ว ฟังก์ชัน constructor ก็เป็นแค่ฟังก์ชันธรรมดา แต่มีข้อตกลงร่วมกันอยู่สองข้อ:

1. ชื่อฟังก์ชันจะขึ้นต้นด้วยตัวพิมพ์ใหญ่เสมอ
2. ควรเรียกใช้งานผ่านตัวดำเนินการ `"new"` เท่านั้น

ตัวอย่างเช่น:

```js run
function User(name) {
  this.name = name;
  this.isAdmin = false;
}

*!*
let user = new User("Jack");
*/!*

alert(user.name); // Jack
alert(user.isAdmin); // false
```

พอเรียกฟังก์ชันผ่าน `new` จะเกิดขั้นตอนดังนี้:

1. สร้างออบเจ็กต์เปล่าๆ ขึ้นมาใหม่ แล้วกำหนดให้เป็น `this`
2. รันบอดี้ของฟังก์ชัน ซึ่งปกติจะเพิ่มพร็อพเพอร์ตี้ต่างๆ ลงใน `this`
3. คืนค่า `this` กลับออกมา

พูดง่ายๆ ก็คือ `new User(...)` ทำงานคล้ายๆ กับ:

```js
function User(name) {
*!*
  // this = {};  (โดยนัยยะ)
*/!*

  // เพิ่มพร็อพเพอร์ตี้ลงใน this
  this.name = name;
  this.isAdmin = false;

*!*
  // return this;  (โดยนัยยะ)
*/!*
}
```

ดังนั้น `let user = new User("Jack")` จะให้ผลลัพธ์เหมือนกับ:

```js
let user = {
  name: "Jack",
  isAdmin: false
};
```

ทีนี้ถ้าต้องการสร้าง user คนอื่น ก็แค่เรียก `new User("Ann")`, `new User("Alice")` — สั้นกว่าและอ่านง่ายกว่าการเขียน object literal ซ้ำๆ มาก

นี่แหละคือจุดประสงค์หลักของ constructor — ทำให้โค้ดสร้างออบเจ็กต์นำกลับมาใช้ซ้ำได้ง่าย

ขอย้ำอีกครั้ง — ฟังก์ชันอะไรก็ตาม (ยกเว้น arrow function ที่ไม่มี `this`) สามารถใช้เป็น constructor ได้ แค่เรียกผ่าน `new` ก็จะทำงานตามขั้นตอนข้างต้นทันที ส่วนการตั้งชื่อขึ้นต้นด้วยตัวพิมพ์ใหญ่เป็นแค่ข้อตกลงร่วมกัน เพื่อบอกว่าฟังก์ชันนี้ควรเรียกผ่าน `new`

````smart header="new function() { ... }"
ถ้าต้องสร้างออบเจ็กต์ที่ซับซ้อนและมีโค้ดหลายบรรทัด เราครอบด้วย constructor function แล้วเรียกใช้ทันทีก็ได้ แบบนี้:

```js
// สร้างฟังก์ชันแล้วเรียกผ่าน new ทันที
let user = new function() {
  this.name = "John";
  this.isAdmin = false;

  // ...โค้ดอื่นๆ ในการสร้าง user
  // อาจมีตรรกะและคำสั่งที่ซับซ้อน
  // มีตัวแปรภายใน ฯลฯ
};
```

Constructor แบบนี้เรียกซ้ำอีกไม่ได้ เพราะไม่ได้เก็บไว้ที่ไหนเลย — สร้างแล้วเรียกใช้ทันทีจบ เทคนิคนี้เหมาะกับกรณีที่ต้องการสร้างออบเจ็กต์ซับซ้อนตัวเดียว โดยไม่ต้องนำไปใช้ซ้ำในอนาคต
````

## ทดสอบโหมด Constructor: new.target

```smart header="เนื้อหาขั้นสูง"
ไวยากรณ์ในส่วนนี้ไม่ค่อยได้ใช้ ข้ามไปก่อนได้เลยถ้ายังไม่อยากลงลึก
```

ภายในฟังก์ชัน เราตรวจสอบได้ว่าถูกเรียกผ่าน `new` หรือไม่ โดยใช้พร็อพเพอร์ตี้พิเศษ `new.target`

ถ้าเรียกแบบปกติ `new.target` จะเป็น `undefined` แต่ถ้าเรียกผ่าน `new` จะได้ตัวฟังก์ชันนั้นกลับมา:

```js run
function User() {
  alert(new.target);
}

// ไม่ใช้ "new":
*!*
User(); // undefined
*/!*

// ใช้ "new":
*!*
new User(); // function User { ... }
*/!*
```

ใช้ตรวจสอบภายในฟังก์ชันได้ว่า ถูกเรียกในโหมด "constructor" (ผ่าน `new`) หรือโหมด "ปกติ" นั่นเอง

นอกจากนี้ยังทำให้การเรียกแบบมีและไม่มี `new` ทำงานเหมือนกันได้ด้วย เช่น:

```js run
function User(name) {
  if (!new.target) { // ถ้าเรียกโดยไม่มี new
    return new User(name); // ...จะใส่ new ให้เอง
  }

  this.name = name;
}

let john = User("John"); // เปลี่ยนทางไปเรียกใช้ new User
alert(john.name); // John
```

วิธีนี้พบได้ในไลบรารีบางตัวที่ต้องการให้ไวยากรณ์ยืดหยุ่น — จะเรียกด้วย `new` หรือไม่ก็ได้ ทำงานเหมือนกัน

แต่ไม่ควรใช้ทุกที่นะ เพราะการละ `new` ทำให้โค้ดคลุมเครือ — พอเห็น `new` ทุกคนจะเข้าใจทันทีว่ากำลังสร้างออบเจ็กต์ใหม่

## การ return จาก constructor

โดยปกติ constructor จะไม่มีคำสั่ง `return` เพราะหน้าที่ของมันคือเขียนข้อมูลลงใน `this` แล้ว `this` ก็จะกลายเป็นผลลัพธ์โดยอัตโนมัติ

แต่ถ้ามีคำสั่ง `return` อยู่ กฎง่ายๆ คือ:

- ถ้า `return` ออบเจ็กต์ — จะได้ออบเจ็กต์นั้นแทน `this`
- ถ้า `return` ค่า primitive — จะถูกเพิกเฉย

พูดสั้นๆ ก็คือ `return` พร้อมออบเจ็กต์จะคืนออบเจ็กต์นั้นกลับมา แต่กรณีอื่นทั้งหมดจะคืน `this` เสมอ

เช่น ในตัวอย่างนี้ `return` เขียนทับ `this` โดยคืนออบเจ็กต์อื่นออกมา:

```js run
function BigUser() {

  this.name = "John";

  return { name: "Godzilla" };  // <-- คืนออบเจ็กต์นี้
}

alert( new BigUser().name );  // Godzilla, ได้ออบเจ็กต์ที่ return ออกมา
```

ส่วนในนี้คือตัวอย่างการ `return` เปล่าๆ (หรืออาจใส่ค่าพื้นฐานหลังจากนั้นก็ได้ ไม่สำคัญ):

```js run
function SmallUser() {

  this.name = "John";

  return; // <-- คืนค่า this
}

alert( new SmallUser().name );  // John
```

ปกติแล้ว constructor ไม่มีคำสั่ง `return` อยู่แล้ว ที่พูดถึงพฤติกรรมตรงนี้ก็เพื่อให้เข้าใจกลไกอย่างครบถ้วนนั่นเอง

````smart header="ละวงเล็บได้"
ถ้าไม่มีอาร์กิวเมนต์ เราละวงเล็บหลัง `new` ได้:

```js
let user = new User; // <-- ไม่มีวงเล็บ
// เหมือนกับ
// let user = new User();
```

การละวงเล็บแบบนี้ไม่ถือว่าเป็น "แบบแผนที่ดี" แต่ไวยากรณ์ของภาษาอนุญาตให้ทำได้
````

## เมธอดใน constructor

ฟังก์ชัน constructor ช่วยให้การสร้างออบเจ็กต์ยืดหยุ่นมาก เราสามารถกำหนดพารามิเตอร์ที่ระบุว่าจะสร้างออบเจ็กต์แบบไหน และจะใส่อะไรลงไป

แน่นอนว่านอกจากพร็อพเพอร์ตี้แล้ว เรายังเพิ่มเมธอดลงใน `this` ได้ด้วย

ตัวอย่างเช่น `new User(name)` ด้านล่างจะสร้างออบเจ็กต์ที่มี `name` และเมธอด `sayHi`:

```js run
function User(name) {
  this.name = name;

  this.sayHi = function() {
    alert( "My name is: " + this.name );
  };
}

*!*
let john = new User("John");

john.sayHi(); // My name is: John
*/!*

/*
john = {
   name: "John",
   sayHi: function() { ... }
}
*/
```

สำหรับการสร้างออบเจ็กต์ที่ซับซ้อนกว่านี้ จะมีไวยากรณ์ขั้นสูงอย่าง [classes](info:classes) ซึ่งเราจะได้เรียนกันในบทหลังๆ

## สรุป

- ฟังก์ชัน constructor (เรียกสั้นๆ ว่า constructor) ก็คือฟังก์ชันปกติ แต่มีข้อตกลงว่าชื่อต้องขึ้นต้นด้วยตัวพิมพ์ใหญ่
- ควรเรียกผ่านตัวดำเนินการ `new` เท่านั้น ซึ่งจะสร้าง `this` เปล่าๆ ขึ้นมาตอนเริ่ม แล้วคืน `this` ที่มีข้อมูลกลับไปตอนจบ

เราใช้ constructor ในการสร้างออบเจ็กต์ที่มีลักษณะคล้ายๆ กันหลายตัว

JavaScript เองก็มี constructor สำหรับออบเจ็กต์ที่ติดมากับตัวภาษา เช่น `Date` สำหรับวันที่, `Set` สำหรับเซ็ต และอื่นๆ อีกมากมายที่เราจะได้เรียนกันในบทต่อๆ ไป

````smart header="ออบเจ็กต์ เรามีอะไรต้องคุยอีกมาก!"
ในบทนี้เราเรียนรู้แค่พื้นฐานเกี่ยวกับออบเจ็กต์และ constructor ซึ่งจำเป็นสำหรับทำความเข้าใจเรื่องชนิดข้อมูลและฟังก์ชันในบทถัดๆ ไป

พอเราคุ้นเคยกับพื้นฐานเหล่านี้แล้ว จะกลับมาศึกษาออบเจ็กต์แบบเจาะลึกอีกครั้งในบท <info:prototypes> และ <info:classes>
````
