# การแปลงออบเจ็กต์เป็นค่าพื้นฐาน (primitive)

เกิดอะไรขึ้นเมื่อเรานำออบเจ็กต์ไปบวก (`obj1 + obj2`) ลบ (`obj1 - obj2`) หรือแสดงผลด้วย `alert(obj)`?

ในกรณีเหล่านี้ ออบเจ็กต์จะถูกแปลงเป็นค่า primitive โดยอัตโนมัติ จากนั้นตัวดำเนินการจึงทำงานกับค่า primitive ที่ได้

จากบท <info:type-conversions> เราได้เรียนรู้กฎการแปลงค่าตัวเลข สตริง และบูลีนของค่า primitive ไปแล้ว แต่ยังเว้นเรื่องออบเจ็กต์ไว้ ทีนี้พอเรารู้เรื่องเมธอดและ symbol แล้ว ก็ถึงเวลาเติมเต็มส่วนที่ขาดไป

1. ออบเจ็กต์ทุกตัวเป็น `true` ในบริบทบูลีน — ง่ายๆ แค่นั้น จึงมีแค่การแปลงเป็นตัวเลขและสตริงเท่านั้น
2. การแปลงเป็นตัวเลขจะเกิดขึ้นเมื่อเราลบออบเจ็กต์หรือใช้ฟังก์ชันทางคณิตศาสตร์ เช่น ออบเจ็กต์ `Date` (จะเรียนในบท <info:date>) สามารถนำมาลบกันได้ โดยผลลัพธ์ของ `date1 - date2` คือระยะเวลาต่างกันระหว่างสองวัน
3. ส่วนการแปลงเป็นสตริง มักเกิดขึ้นเมื่อแสดงผลออบเจ็กต์ด้วย `alert(obj)` หรือในสถานการณ์คล้ายๆ กัน

เรากำหนดเองได้ว่าจะให้ออบเจ็กต์แปลงเป็นสตริงหรือตัวเลขอย่างไร โดยใช้เมธอดพิเศษของออบเจ็กต์

ทีนี้มาดูรายละเอียดทางเทคนิคกัน เพราะเป็นทางเดียวที่จะเข้าใจหัวข้อนี้อย่างลึกซึ้ง

## Hint

JavaScript ตัดสินใจอย่างไรว่าจะแปลงแบบไหน?

การแปลงมี 3 รูปแบบ เรียกว่า "hint" ตามที่อธิบายไว้ใน [สเปค](https://tc39.github.io/ecma262/#sec-toprimitive):

`"string"`
: ใช้เมื่อต้องแปลงออบเจ็กต์เป็นสตริง เช่น ตอนแสดงผลหรือใช้เป็น key ของพร็อพเพอร์ตี้:

    ```js
    // output
    alert(obj);

    // ใช้ออบเจ็กต์เป็น key ของพร็อพเพอร์ตี้
    anotherObj[obj] = 123;
    ```

`"number"`
: ใช้เมื่อต้องแปลงเป็นตัวเลข เช่น ในการคำนวณทางคณิตศาสตร์:

    ```js
    // แปลงเป็นตัวเลขโดยตรง
    let num = Number(obj);

    // การคำนวณ (ยกเว้น binary plus)
    let n = +obj; // unary plus
    let delta = date1 - date2;

    // เปรียบเทียบมากกว่า/น้อยกว่า
    let greater = user1 > user2;
    ```

    ฟังก์ชันคณิตศาสตร์ในตัว (built-in) ส่วนใหญ่ก็ใช้การแปลงแบบนี้ด้วย

`"default"`
: เกิดขึ้นน้อยครั้ง เมื่อตัวดำเนินการ "ไม่แน่ใจ" ว่าต้องการข้อมูลประเภทไหน

    เช่น ตัวดำเนินการบวก `+` ใช้ได้ทั้งกับสตริง (ต่อกัน) และตัวเลข (บวกกัน) ดังนั้นถ้าได้รับออบเจ็กต์มา จึงใช้ hint `"default"` ในการแปลง

    นอกจากนี้ ถ้าเปรียบเทียบออบเจ็กต์ด้วย `==` กับสตริง ตัวเลข หรือ symbol ก็ไม่ชัดเจนว่าควรแปลงแบบไหน จึงใช้ hint `"default"` เช่นกัน

    ```js
    // binary plus ใช้ hint "default"
    let total = obj1 + obj2;

    // obj == number ใช้ hint "default"
    if (user == 1) { ... };
    ```

    ส่วนตัวดำเนินการเปรียบเทียบ `<` `>` แม้จะใช้ได้ทั้งกับสตริงและตัวเลข แต่กลับใช้ hint `"number"` ไม่ใช่ `"default"` — ด้วยเหตุผลทางประวัติศาสตร์

    ในทางปฏิบัติ ไม่ต้องจำรายละเอียดพวกนี้ทั้งหมดก็ได้ เพราะออบเจ็กต์ในตัว (built-in) ทุกตัว ยกเว้น `Date` จะใช้การแปลงแบบ `"default"` เหมือนกับ `"number"` อยู่แล้ว เราก็ควรทำแบบเดียวกัน

```smart header="ไม่มี hint `\"boolean\"` นะ"
สังเกตว่า hint มีแค่ 3 แบบเท่านั้น ง่ายๆ แค่นี้เลย

ไม่มี hint "boolean" (เพราะออบเจ็กต์ทุกตัวเป็น `true` ในบริบทบูลีนอยู่แล้ว) ถ้าเราจัดการ `"default"` กับ `"number"` เหมือนกัน เหมือนที่ built-in ส่วนใหญ่ทำ ก็จะเหลือแค่ 2 แบบเท่านั้นเอง
```

**เมื่อต้องแปลงออบเจ็กต์เป็นค่า primitive JavaScript จะพยายามเรียกเมธอดตามลำดับนี้:**

1. เรียก `obj[Symbol.toPrimitive](hint)` — ถ้ามีเมธอดนี้อยู่ โดย `Symbol.toPrimitive` เป็น system symbol
2. ถ้าไม่มี และ hint เป็น `"string"`
    - ลองเรียก `obj.toString()` หรือ `obj.valueOf()` เลือกตัวที่มีอยู่
3. ถ้าไม่มี และ hint เป็น `"number"` หรือ `"default"`
    - ลองเรียก `obj.valueOf()` หรือ `obj.toString()` เลือกตัวที่มีอยู่

## Symbol.toPrimitive

มาเริ่มจากวิธีแรกกัน มี symbol ในตัวชื่อ `Symbol.toPrimitive` ที่ใช้ตั้งชื่อเมธอดสำหรับแปลงออบเจ็กต์ หน้าตาเป็นแบบนี้:

```js
obj[Symbol.toPrimitive] = function(hint) {
  // ต้องคืนค่า primitive
  // hint จะเป็นหนึ่งใน "string", "number", "default"
};
```

ลองดูตัวอย่าง ออบเจ็กต์ `user` นี้ใช้ `Symbol.toPrimitive`:

```js run
let user = {
  name: "John",
  money: 1000,

  [Symbol.toPrimitive](hint) {
    alert(`hint: ${hint}`);
    return hint == "string" ? `{name: "${this.name}"}` : this.money;
  }
};

// ตัวอย่างการแปลง:
alert(user); // hint: string -> {name: "John"}
alert(+user); // hint: number -> 1000
alert(user + 500); // hint: default -> 1500
```

จะเห็นว่า `user` กลายเป็นสตริงอธิบายตัวเองหรือจำนวนเงิน ขึ้นอยู่กับรูปแบบการแปลง โดยใช้แค่เมธอดเดียว `user[Symbol.toPrimitive]` จัดการได้ทุกกรณีเลย


## toString/valueOf

เมธอด `toString` และ `valueOf` มีมาตั้งแต่สมัยเก่า ก่อนที่จะมี symbol ด้วยซ้ำ เป็นเมธอดที่ใช้ชื่อ string ธรรมดา ถือเป็นวิธีแปลงแบบ "ดั้งเดิม" นั่นเอง

ถ้าไม่มี `Symbol.toPrimitive` JavaScript จะมองหาเมธอดเหล่านี้แทน ตามลำดับ:

- สำหรับ hint `"string"`: เรียก `toString` ก่อน ถ้าไม่มีหรือคืนค่าเป็นออบเจ็กต์ จึงไปเรียก `valueOf` (ดังนั้น `toString` จึงมีลำดับก่อนสำหรับการแปลงเป็นสตริง)
- สำหรับ hint อื่นๆ: เรียก `valueOf` ก่อน ถ้าไม่มีหรือคืนค่าเป็นออบเจ็กต์ จึงไปเรียก `toString` (ดังนั้น `valueOf` จึงมีลำดับก่อนสำหรับการคำนวณ)

เมธอดเหล่านี้ต้องคืนค่า primitive ถ้า `toString` หรือ `valueOf` คืนออบเจ็กต์ ก็จะถูกข้ามไปเลย เหมือนไม่มีเมธอดนั้นอยู่

ออบเจ็กต์ทั่วไปมี `toString` และ `valueOf` ค่าเริ่มต้นดังนี้:

- `toString` คืนสตริง `"[object Object]"`
- `valueOf` คืนออบเจ็กต์ตัวมันเอง

ลองดู:

```js run
let user = {name: "John"};

alert(user); // [object Object]
alert(user.valueOf() === user); // true
```

ดังนั้น ถ้าใช้ออบเจ็กต์ในตำแหน่งที่คาดหวังสตริง เช่น `alert` จะเห็น `[object Object]` เป็นค่าเริ่มต้น

ส่วน `valueOf` ที่คืนออบเจ็กต์ตัวมันเองนั้น กล่าวถึงไว้เพื่อความครบถ้วนเท่านั้น เพราะคืนออบเจ็กต์กลับมาก็เท่ากับถูกข้ามไป ไม่ต้องถามว่าทำไมถึงเป็นแบบนี้ — เหตุผลทางประวัติศาสตร์ล้วนๆ ถือว่าไม่มีก็ได้

ทีนี้มาลองเขียนเมธอดเหล่านี้เองกัน

ตัวอย่าง `user` นี้ทำงานเหมือนตัวอย่างก่อนหน้า แต่ใช้ `toString` กับ `valueOf` แทน `Symbol.toPrimitive`:

```js run
let user = {
  name: "John",
  money: 1000,

  // สำหรับ hint "string"
  toString() {
    return `{name: "${this.name}"}`;
  },

  // สำหรับ hint "number" หรือ "default"
  valueOf() {
    return this.money;
  }

};

alert(user); // toString -> {name: "John"}
alert(+user); // valueOf -> 1000
alert(user + 500); // valueOf -> 1500
```

ผลลัพธ์เหมือนกับตัวอย่าง `Symbol.toPrimitive` ก่อนหน้าเลย

บ่อยครั้งที่เราอยากจัดการการแปลงทุกแบบไว้ที่เดียว กรณีนี้เขียนแค่ `toString` ตัวเดียวก็พอ:

```js run
let user = {
  name: "John",

  toString() {
    return this.name;
  }
};

alert(user); // toString -> John
alert(user + 500); // toString -> John500
```

ถ้าไม่มี `Symbol.toPrimitive` และ `valueOf` เมธอด `toString` จะรับหน้าที่แปลงออบเจ็กต์เป็นค่า primitive ทั้งหมดเอง

### ชนิดของค่าที่คืน

สิ่งสำคัญที่ต้องรู้คือ เมธอดที่ใช้แปลงออบเจ็กต์ไม่จำเป็นต้องคืนค่า primitive ตรงกับ hint เสมอไป

ไม่มีข้อบังคับว่า `toString` ต้องคืนสตริง หรือ `Symbol.toPrimitive` ต้องคืนตัวเลขเมื่อ hint เป็น `"number"`

สิ่งเดียวที่บังคับคือ — ต้องคืนค่า primitive ไม่ใช่ออบเจ็กต์

```smart header="หมายเหตุทางประวัติศาสตร์"
ในอดีต ถ้า `toString` หรือ `valueOf` คืนออบเจ็กต์ จะไม่เกิด error แต่ค่านั้นจะถูกข้ามไปเฉยๆ (เหมือนไม่มีเมธอดนั้นอยู่) เพราะ JavaScript สมัยนั้นยังไม่มีแนวคิดเรื่อง "error" ที่ดีพอ

แต่ `Symbol.toPrimitive` เข้มงวดกว่า — *ต้อง*คืนค่า primitive เท่านั้น ไม่งั้นจะเกิด error
```

## การแปลงต่อเนื่อง

ดังที่ทราบกันแล้ว ตัวดำเนินการและฟังก์ชันหลายตัวจะแปลงชนิดข้อมูลให้ เช่น การคูณ `*` จะแปลงตัวถูกดำเนินการเป็นตัวเลข

ถ้าส่งออบเจ็กต์เป็นอาร์กิวเมนต์ การคำนวณจะเกิดขึ้น 2 ขั้นตอน:
1. แปลงออบเจ็กต์เป็นค่า primitive ก่อน (ตามกฎข้างต้น)
2. ถ้าค่า primitive ที่ได้ยังไม่ตรงชนิด ก็จะแปลงอีกครั้ง

ตัวอย่าง:

```js run
let obj = {
  // toString จัดการการแปลงทั้งหมดเมื่อไม่มีเมธอดอื่น
  toString() {
    return "2";
  }
};

alert(obj * 2); // 4, ออบเจ็กต์แปลงเป็น primitive ได้ "2" แล้ว "2" ถูกคูณจนกลายเป็น 4
```

1. การคูณ `obj * 2` แปลง `obj` เป็นค่า primitive ก่อน ได้สตริง `"2"` (จาก `toString`)
2. จากนั้น `"2" * 2` กลายเป็น `2 * 2` (สตริงถูกแปลงเป็นตัวเลข)

ตัวดำเนินการบวก `+` ก็ต่อสตริงได้ด้วย จึงได้ผลลัพธ์ต่างออกไป:

```js run
let obj = {
  toString() {
    return "2";
  }
};

alert(obj + 2); // "22" ("2" + 2), แปลงเป็น primitive ได้สตริง => ต่อสตริงแทน
```

## สรุป

ฟังก์ชันและตัวดำเนินการในตัวหลายตัวที่คาดหวังค่า primitive จะเรียกการแปลงออบเจ็กต์โดยอัตโนมัติ

การแปลงมี 3 แบบ (hint):
- `"string"` (สำหรับ `alert` และตัวดำเนินการที่ต้องการสตริง)
- `"number"` (สำหรับการคำนวณ)
- `"default"` (สำหรับตัวดำเนินการบางตัว ที่ไม่แน่ใจว่าต้องการชนิดไหน)

สเปคระบุชัดเจนว่าตัวดำเนินการแต่ละตัวใช้ hint ไหน ตัวดำเนินการที่ "ไม่แน่ใจ" ว่าต้องการชนิดไหนจริงๆ มีน้อยมาก และสำหรับ built-in ส่วนใหญ่ hint `"default"` ก็ถูกจัดการเหมือน `"number"` อยู่แล้ว จึงมักรวมสองแบบนี้เข้าด้วยกัน

ลำดับขั้นตอนการแปลง:

1. เรียก `obj[Symbol.toPrimitive](hint)` ถ้ามีเมธอดนี้อยู่
2. ถ้าไม่มี และ hint เป็น `"string"`
    - ลองเรียก `obj.toString()` หรือ `obj.valueOf()` เลือกตัวที่มีอยู่
3. ถ้าไม่มี และ hint เป็น `"number"` หรือ `"default"`
    - ลองเรียก `obj.valueOf()` หรือ `obj.toString()` เลือกตัวที่มีอยู่

ในทางปฏิบัติ แค่เขียน `obj.toString()` ตัวเดียวก็มักจะเพียงพอแล้ว เมธอดนี้จะจัดการการแปลงทุกแบบ และมักใช้สำหรับแสดงข้อมูลออบเจ็กต์ในรูปแบบที่อ่านง่าย ไม่ว่าจะเป็นการ log หรือดีบั๊ก
