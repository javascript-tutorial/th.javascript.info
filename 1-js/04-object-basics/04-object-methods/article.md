# เมธอดของออบเจ็กต์ และ "this"

เรามักสร้างออบเจ็กต์ขึ้นมาเพื่อแทนสิ่งต่างๆ ในโลกจริง เช่น ผู้ใช้ ใบสั่งซื้อ และอื่นๆ:

```js
let user = {
  name: "John",
  age: 30
};
```

ในโลกจริง ผู้ใช้ *ทำ* สิ่งต่างๆ ได้ เช่น เลือกสินค้าจากตะกร้า เข้าสู่ระบบ ออกจากระบบ ฯลฯ

ใน JavaScript การกระทำเหล่านี้แทนด้วยฟังก์ชันที่อยู่ในพร็อพเพอร์ตี้ของออบเจ็กต์

## ตัวอย่างเมธอด

เริ่มจากมาสอนให้ `user` ทักทายกันก่อน:

```js run
let user = {
  name: "John",
  age: 30
};

*!*
user.sayHi = function() {
  alert("Hello!");
};
*/!*

user.sayHi(); // Hello!
```

ตรงนี้เราใช้ Function Expression สร้างฟังก์ชันขึ้นมา แล้วกำหนดให้กับพร็อพเพอร์ตี้ `user.sayHi` ของออบเจ็กต์

จากนั้นก็เรียกด้วย `user.sayHi()` — ตอนนี้ผู้ใช้พูดได้แล้ว!

ฟังก์ชันที่เป็นพร็อพเพอร์ตี้ของออบเจ็กต์ เราเรียกว่า *เมธอด*

ในตัวอย่างนี้ เราจึงมีเมธอด `sayHi` ของออบเจ็กต์ `user`

นอกจากนี้ยังใช้ฟังก์ชันที่ประกาศไว้ก่อนหน้ามาเป็นเมธอดได้ด้วย เช่น:

```js run
let user = {
  // ...
};

*!*
// ประกาศฟังก์ชันก่อน
function sayHi() {
  alert("Hello!");
}

// จากนั้นเพิ่มเป็นเมธอด
user.sayHi = sayHi;
*/!*

user.sayHi(); // Hello!
```

```smart header="การเขียนโปรแกรมเชิงวัตถุ"
เมื่อเราเขียนโค้ดโดยใช้ออบเจ็กต์แทนสิ่งต่างๆ ในโลกจริง แนวคิดนี้เรียกว่า [การเขียนโปรแกรมเชิงวัตถุ](https://en.wikipedia.org/wiki/Object-oriented_programming) หรือเรียกสั้นๆ ว่า "OOP"

OOP เป็นศาสตร์ที่กว้างและน่าสนใจมาก — จะเลือกออบเจ็กต์อะไรดี? จะจัดการปฏิสัมพันธ์ระหว่างออบเจ็กต์ยังไง? ทั้งหมดนี้คือเรื่องของสถาปัตยกรรม ซึ่งมีหนังสือดีๆ มากมาย เช่น "Design Patterns: Elements of Reusable Object-Oriented Software" โดย E. Gamma, R. Helm, R. Johnson, J. Vissides หรือ "Object-Oriented Analysis and Design with Applications" โดย G. Booch เป็นต้น
```

### เมธอดแบบย่อ

มีวิธีเขียนเมธอดใน object literal ให้สั้นลงได้:

```js
// ออบเจ็กต์เหล่านี้ทำงานเหมือนกัน

user = {
  sayHi: function() {
    alert("Hello");
  }
};

// เมธอดแบบย่อดูดีกว่าใช่ไหม?
user = {
*!*
  sayHi() { // เหมือนกับ "sayHi: function(){...}"
*/!*
    alert("Hello");
  }
};
```

จะเห็นว่าเราละ `"function"` ทิ้งไปได้เลย แล้วเขียนแค่ `sayHi()`

จริงๆ แล้วสองแบบนี้ไม่ได้เหมือนกันซะทีเดียว มีความแตกต่างเล็กน้อยที่เกี่ยวข้องกับการสืบทอดของออบเจ็กต์ (จะกล่าวถึงทีหลัง) แต่ตอนนี้ยังไม่มีผล เกือบทุกกรณีแบบย่อจะเป็นที่นิยมมากกว่า

## "this" ในเมธอด

บ่อยครั้งที่เมธอดต้องเข้าถึงข้อมูลภายในออบเจ็กต์เพื่อทำงาน

เช่น โค้ดภายใน `user.sayHi()` อาจต้องใช้ชื่อของ `user`

**เมธอดเข้าถึงออบเจ็กต์ได้ผ่านคีย์เวิร์ด `this`**

ค่าของ `this` คือออบเจ็กต์ที่อยู่ "ก่อนจุด" ซึ่งเป็นตัวที่เรียกเมธอด

เช่น:

```js run
let user = {
  name: "John",
  age: 30,

  sayHi() {
*!*
    // "this" คือออบเจ็กต์ "ปัจจุบัน"
    alert(this.name);
*/!*
  }

};

user.sayHi(); // John
```

ขณะที่ `user.sayHi()` ทำงาน ค่าของ `this` จะเป็น `user`

ในทางเทคนิคแล้ว เราเข้าถึงออบเจ็กต์ได้โดยไม่ต้องใช้ `this` ก็ได้ — อ้างอิงผ่านตัวแปรภายนอกตรงๆ แบบนี้:

```js
let user = {
  name: "John",
  age: 30,

  sayHi() {
*!*
    alert(user.name); // ใช้ "user" แทน "this"
*/!*
  }

};
```

...แต่โค้ดแบบนี้ไม่น่าเชื่อถือ ถ้าเราคัดลอก `user` ไปยังตัวแปรอื่น เช่น `admin = user` แล้วเขียนทับ `user` ด้วยค่าอื่น ก็จะเข้าถึงออบเจ็กต์ผิดตัวทันที

ลองดูตัวอย่าง:

```js run
let user = {
  name: "John",
  age: 30,

  sayHi() {
*!*
    alert( user.name ); // เกิด error!
*/!*
  }

};


let admin = user;
user = null; // เขียนทับเพื่อให้เห็นชัดเจน

*!*
admin.sayHi(); // TypeError: Cannot read property 'name' of null
*/!*
```

ถ้าเราใช้ `this.name` แทน `user.name` ภายใน `alert` โค้ดก็จะทำงานได้ถูกต้อง

## "this" ไม่ได้ถูกผูกไว้

ใน JavaScript คีย์เวิร์ด `this` ทำงานไม่เหมือนภาษาโปรแกรมอื่นๆ ส่วนใหญ่ เราใช้ `this` ในฟังก์ชันไหนก็ได้ แม้ไม่ใช่เมธอดของออบเจ็กต์ก็ตาม

ตัวอย่างนี้ไม่มี syntax error:

```js
function sayHi() {
  alert( *!*this*/!*.name );
}
```

ค่าของ `this` จะถูกประเมินตอน run-time ขึ้นอยู่กับบริบท (context)

ลองดูตัวอย่าง — ฟังก์ชันเดียวกันถูกกำหนดให้กับออบเจ็กต์สองตัว แต่ `this` ในแต่ละการเรียกกลับต่างกัน:

```js run
let user = { name: "John" };
let admin = { name: "Admin" };

function sayHi() {
  alert( this.name );
}

*!*
// ใช้ฟังก์ชันเดียวกันในออบเจ็กต์สองตัว
user.f = sayHi;
admin.f = sayHi;
*/!*

// การเรียกเหล่านี้มี this ต่างกัน
// "this" ภายในฟังก์ชันคือออบเจ็กต์ "ก่อนจุด"
user.f(); // John  (this == user)
admin.f(); // Admin  (this == admin)

admin['f'](); // Admin (การเข้าถึงด้วยจุดหรือวงเล็บก็ไม่ต่างกัน)
```

กฎง่ายๆ คือ ถ้าเรียก `obj.f()` แล้ว `this` ภายในฟังก์ชัน `f` จะเป็น `obj` — ในตัวอย่างข้างต้นก็คือ `user` หรือ `admin` นั่นเอง

````smart header="เรียกฟังก์ชันโดยไม่มีออบเจ็กต์: `this == undefined`"
เราเรียกฟังก์ชันโดยไม่มีออบเจ็กต์เลยก็ได้:

```js run
function sayHi() {
  alert(this);
}

sayHi(); // undefined
```

ในกรณีนี้ `this` จะเป็น `undefined` ใน strict mode ถ้าพยายามเข้าถึง `this.name` ก็จะเกิด error

ส่วนใน non-strict mode ค่าของ `this` จะเป็น *global object* (คือ `window` ในเบราว์เซอร์ — จะได้เรียนเพิ่มในบท [](info:global-object)) นี่เป็นพฤติกรรมดั้งเดิมที่ `"use strict"` แก้ไข

โดยปกติแล้วการเรียกแบบนี้ถือเป็น bug เพราะถ้าฟังก์ชันมีการใช้ `this` ก็หมายความว่าควรเรียกในบริบทของออบเจ็กต์
````

````smart header="ผลของ `this` ที่ไม่ถูกผูกไว้"
ถ้ามาจากภาษาโปรแกรมอื่น อาจคุ้นเคยกับแนวคิด "`this` ที่ถูกผูกไว้แล้ว" — คือเมธอดที่ประกาศในออบเจ็กต์จะมี `this` อ้างถึงออบเจ็กต์นั้นเสมอ

แต่ใน JavaScript `this` เป็นอิสระ ค่าจะถูกประเมินตอนเรียก (call-time) ไม่ได้ขึ้นอยู่กับว่าเมธอดประกาศที่ไหน แต่ขึ้นกับว่าออบเจ็กต์ตัวไหนอยู่ "ก่อนจุด"

การที่ `this` ถูกประเมินตอน run-time มีทั้งข้อดีและข้อเสีย — ข้อดีคือฟังก์ชันเดียวนำไปใช้ซ้ำกับออบเจ็กต์ต่างๆ ได้ แต่ข้อเสียคือยืดหยุ่นมากก็มีโอกาสพลาดมากขึ้นด้วย

เราไม่ได้จะตัดสินว่าการออกแบบแบบนี้ดีหรือไม่ดี แต่จะมาเรียนรู้วิธีใช้งานให้ถูกต้อง — ใช้ประโยชน์จากมันให้ได้ และหลีกเลี่ยงปัญหา
````

## Arrow function ไม่มี "this"

Arrow function มีลักษณะพิเศษอย่างหนึ่ง — ไม่มี `this` เป็นของตัวเอง ถ้าอ้างถึง `this` ภายใน arrow function จะดึงค่ามาจากฟังก์ชัน "ปกติ" ที่ครอบอยู่ด้านนอกแทน

เช่น ในตัวอย่างนี้ `arrow()` ใช้ `this` จากเมธอด `user.sayHi()` ด้านนอก:

```js run
let user = {
  firstName: "Ilya",
  sayHi() {
    let arrow = () => alert(this.firstName);
    arrow();
  }
};

user.sayHi(); // Ilya
```

นี่เป็นฟีเจอร์พิเศษของ arrow function ซึ่งมีประโยชน์มากเวลาที่เราไม่ต้องการ `this` แยกต่างหาก แต่ต้องการใช้ `this` จากบริบทภายนอก เราจะเจาะลึกเรื่อง arrow function กันอีกทีในบท <info:arrow-functions>

## สรุป

- ฟังก์ชันที่เก็บไว้ในพร็อพเพอร์ตี้ของออบเจ็กต์ เรียกว่า "เมธอด"
- เมธอดทำให้ออบเจ็กต์ "ทำอะไรบางอย่าง" ได้ เช่น `object.doSomething()`
- เมธอดอ้างอิงออบเจ็กต์ได้ผ่าน `this`

ค่าของ `this` ถูกกำหนดตอน run-time:
- ตอนประกาศฟังก์ชัน จะใช้ `this` ก็ได้ แต่ `this` จะยังไม่มีค่าจนกว่าฟังก์ชันจะถูกเรียก
- ฟังก์ชันสามารถคัดลอกไปใช้กับออบเจ็กต์อื่นได้
- เมื่อเรียกฟังก์ชันในรูปแบบเมธอด `object.method()` ค่าของ `this` ระหว่างการเรียกก็คือ `object`

สิ่งที่ควรรู้คือ arrow function ไม่มี `this` ของตัวเอง — ถ้าเข้าถึง `this` ภายใน arrow function จะดึงค่ามาจากฟังก์ชันด้านนอก
