# สตริง (Strings)

ใน JavaScript ข้อมูลที่เป็นข้อความจะเก็บอยู่ในรูปของสตริง โดยไม่มีชนิดข้อมูลแยกสำหรับอักขระตัวเดียว

รูปแบบภายในของสตริงจะเป็น [UTF-16](https://en.wikipedia.org/wiki/UTF-16) เสมอ ไม่ขึ้นอยู่กับการเข้ารหัสของหน้าเว็บ

## เครื่องหมายคำพูด

มาทบทวนเครื่องหมายคำพูดแต่ละแบบกัน

สตริงสามารถอยู่ภายในเครื่องหมายคำพูดเดี่ยว เครื่องหมายคำพูดคู่ หรือ backticks ก็ได้:

```js
let single = 'single-quoted';
let double = "double-quoted";

let backticks = `backticks`;
```

เครื่องหมายคำพูดเดี่ยวและคู่ทำงานเหมือนกัน แต่ backticks พิเศษกว่าตรงที่แทรกนิพจน์ (expression) ลงในสตริงได้ โดยครอบด้วย `${…}`:

```js run
function sum(a, b) {
  return a + b;
}

alert(`1 + 2 = ${sum(1, 2)}.`); // 1 + 2 = 3.
```

ข้อดีอีกอย่างของ backticks คือเขียนสตริงหลายบรรทัดได้:

```js run
let guestList = `Guests:
 * John
 * Pete
 * Mary
`;

alert(guestList); // a list of guests, multiple lines
```

ดูเป็นธรรมชาติดีใช่ไหม? แต่เครื่องหมายคำพูดเดี่ยวหรือคู่ทำแบบนี้ไม่ได้

ถ้าลองใช้เครื่องหมายคำพูดเดี่ยวหรือคู่แล้วขึ้นบรรทัดใหม่ จะเกิด error:

```js run
let guestList = "Guests: // Error: Unexpected token ILLEGAL
  * John";
```

เครื่องหมายคำพูดเดี่ยวและคู่มีมาตั้งแต่ยุคแรกๆ ของการสร้างภาษา ซึ่งตอนนั้นยังไม่ได้คำนึงถึงสตริงหลายบรรทัด ส่วน backticks มาทีหลัง จึงใช้งานได้หลากหลายกว่า

นอกจากนี้ backticks ยังระบุ "template function" ไว้หน้า backtick แรกได้ด้วย ไวยากรณ์คือ: <code>func&#96;string&#96;</code> โดยฟังก์ชัน `func` จะถูกเรียกอัตโนมัติ รับสตริงและนิพจน์ที่แทรกเข้ามาแล้วประมวลผลได้ ฟีเจอร์นี้เรียกว่า "tagged templates" ไม่ค่อยเจอบ่อยนัก แต่อ่านเพิ่มเติมได้ที่ MDN: [Template literals](mdn:/JavaScript/Reference/Template_literals#Tagged_templates)

## อักขระพิเศษ

จริงๆ แล้วเครื่องหมายคำพูดเดี่ยวและคู่ก็สร้างสตริงหลายบรรทัดได้เหมือนกัน โดยใช้ "อักขระขึ้นบรรทัดใหม่" (newline character) ที่เขียนเป็น `\n`:

```js run
let guestList = "Guests:\n * John\n * Pete\n * Mary";

alert(guestList); // แสดงรายชื่อแขกหลายบรรทัด เหมือนตัวอย่างข้างบน
```

ลองดูตัวอย่างที่ง่ายกว่า — สองบรรทัดนี้ให้ผลลัพธ์เหมือนกัน แค่เขียนคนละแบบ:

```js run
let str1 = "Hello\nWorld"; // สองบรรทัดโดยใช้ "สัญลักษณ์ขึ้นบรรทัดใหม่"

// สองบรรทัดโดยใช้การขึ้นบรรทัดปกติและ backticks
let str2 = `Hello
World`;

alert(str1 == str2); // true แสดงว่าทั้งสองสตริงเหมือนกัน
```

นอกจากนี้ยังมีอักขระพิเศษอื่นๆ ที่พบได้น้อยกว่า:

| อักขระ | คำอธิบาย |
|-----------|-------------|
|`\n`|ขึ้นบรรทัดใหม่|
|`\r`|ในไฟล์ข้อความ Windows จะใช้ `\r\n` สองตัวรวมกันสำหรับขึ้นบรรทัดใหม่ ส่วนระบบอื่นจะใช้แค่ `\n` เหตุผลเป็นเรื่องทางประวัติศาสตร์ ซอฟต์แวร์ Windows ส่วนใหญ่ก็เข้าใจ `\n` เช่นกัน|
|`\'`,&nbsp;`\"`,&nbsp;<code>\\`</code>|เครื่องหมายคำพูด|
|`\\`|แบ็กสแลช|
|`\t`|แท็บ|
|`\b`, `\f`, `\v`| Backspace, Form Feed, Vertical Tab — เป็นอักขระจากยุคเก่า ปัจจุบันแทบไม่ได้ใช้แล้ว (ลืมไปได้เลย)|

จะเห็นว่าอักขระพิเศษทั้งหมดขึ้นต้นด้วยแบ็กสแลช `\` ซึ่งเรียกว่า "escape character"

เนื่องจากแบ็กสแลชเป็นอักขระพิเศษ ถ้าต้องการแสดง `\` จริงๆ ในสตริง ต้องเขียนซ้อนกันสองตัว:

```js run
alert( `The backslash: \\` ); // The backslash: \
```

เครื่องหมายคำพูดที่ "escaped" แล้ว `\'`, `\"`, <code>\\`</code> ใช้สำหรับแทรกเครื่องหมายคำพูดชนิดเดียวกันลงในสตริง

เช่น:

```js run
alert( 'I*!*\'*/!*m the Walrus!' ); // *!*I'm*/!* the Walrus!
```

จะเห็นว่าเราต้องใส่แบ็กสแลชนำหน้า `\'` เพราะถ้าไม่ใส่ JavaScript จะเข้าใจว่าเป็นจุดจบของสตริง

แน่นอนว่าต้อง escape เฉพาะเครื่องหมายที่ตรงกับที่ใช้ครอบสตริงเท่านั้น วิธีที่ง่ายกว่าคือเปลี่ยนไปใช้เครื่องหมายคำพูดอีกแบบหรือ backticks แทน:


```js run
alert( "I'm the Walrus!" ); // I'm the Walrus!
```

นอกจากอักขระพิเศษเหล่านี้แล้ว ยังมีสัญลักษณ์พิเศษสำหรับรหัส Unicode `\u…` ซึ่งไม่ค่อยได้ใช้ อ่านเพิ่มเติมได้ในบทเสริมเรื่อง [Unicode](info:unicode)

## ความยาวของสตริง

พร็อพเพอร์ตี้ `length` บอกความยาวของสตริง:

```js run
alert( `My\n`.length ); // 3
```

สังเกตว่า `\n` เป็น "อักขระพิเศษ" เดียว ดังนั้นความยาวของสตริงนี้จึงเป็น `3`

```warn header="`length` เป็นพร็อพเพอร์ตี้"
คนที่มาจากภาษาอื่นบางทีอาจเผลอเขียน `str.length()` แทน `str.length` ซึ่งใช้ไม่ได้

จำไว้ว่า `str.length` เป็นพร็อพเพอร์ตี้เชิงตัวเลข ไม่ใช่ฟังก์ชัน ไม่ต้องใส่วงเล็บ — ใช้ `.length` ไม่ใช่ `.length()`
```

## การเข้าถึงอักขระ

ถ้าต้องการดึงอักขระที่ตำแหน่ง `pos` ใช้วงเล็บเหลี่ยม `[pos]` หรือเรียกเมธอด [str.at(pos)](mdn:js/String/at) ก็ได้ โดยอักขระแรกเริ่มนับจากตำแหน่งศูนย์:

```js run
let str = `Hello`;

// อักขระแรก
alert( str[0] ); // H
alert( str.at(0) ); // H

// อักขระสุดท้าย
alert( str[str.length - 1] ); // o
alert( str.at(-1) );
```

จะเห็นว่าเมธอด `.at(pos)` มีข้อดีคือใช้ค่าลบได้ ถ้า `pos` เป็นลบ จะนับจากท้ายสตริง

`.at(-1)` คืออักขระตัวสุดท้าย `.at(-2)` คือตัวก่อนสุดท้าย เป็นต้น

ส่วนวงเล็บเหลี่ยมจะคืนค่า `undefined` เสมอเมื่อใส่ค่าลบ เช่น:

```js run
let str = `Hello`;

alert( str[-2] ); // undefined
alert( str.at(-2) ); // l
```

นอกจากนี้ยังวนลูปผ่านอักขระแต่ละตัวด้วย `for..of` ได้:

```js run
for (let char of "Hello") {
  alert(char); // H,e,l,l,o (char becomes "H", then "e", then "l" etc)
}
```

## สตริงเปลี่ยนแปลงไม่ได้

สตริงใน JavaScript เปลี่ยนแปลงไม่ได้ (immutable) — ไม่มีทางแก้ไขอักขระในสตริงที่มีอยู่แล้ว

ลองดูว่าใช้ไม่ได้จริง:

```js run
let str = 'Hi';

str[0] = 'h'; // error
alert( str[0] ); // ไม่ทำงาน
```

วิธีแก้คือสร้างสตริงใหม่ทั้งหมดแล้วกำหนดให้ตัวแปรเดิมแทน:

เช่น:

```js run
let str = 'Hi';

str = 'h' + str[1]; // แทนที่สตริง

alert( str ); // hi
```

ในหัวข้อถัดๆ ไปจะมีตัวอย่างลักษณะนี้อีก

## การเปลี่ยนตัวพิมพ์

เมธอด [toLowerCase()](mdn:js/String/toLowerCase) และ [toUpperCase()](mdn:js/String/toUpperCase) ใช้เปลี่ยนตัวพิมพ์:

```js run
alert( 'Interface'.toUpperCase() ); // INTERFACE
alert( 'Interface'.toLowerCase() ); // interface
```

หรือถ้าต้องการเปลี่ยนแค่ตัวเดียว:

```js run
alert( 'Interface'[0].toLowerCase() ); // 'i'
```

## การค้นหาสตริงย่อย

มีหลายวิธีในการค้นหาสตริงย่อยภายในสตริง

### str.indexOf

เมธอดแรกคือ [str.indexOf(substr, pos)](mdn:js/String/indexOf)

ค้นหา `substr` ภายใน `str` โดยเริ่มจากตำแหน่ง `pos` แล้วคืนค่าตำแหน่งที่พบ หรือ `-1` ถ้าไม่พบ

เช่น:

```js run
let str = 'Widget with id';

alert( str.indexOf('Widget') ); // 0, เพราะ 'Widget' อยู่ตรงจุดเริ่มต้น
alert( str.indexOf('widget') ); // -1, ไม่พบ เพราะคำนึงถึงตัวพิมพ์ใหญ่-เล็ก

alert( str.indexOf("id") ); // 1, "id" พบที่ตำแหน่ง 1 (..idget with id)
```

พารามิเตอร์ตัวที่สองเป็นตัวเลือก ช่วยให้เริ่มค้นหาจากตำแหน่งที่กำหนดได้

เช่น `"id"` ปรากฏครั้งแรกที่ตำแหน่ง `1` ถ้าต้องการหาตำแหน่งถัดไป ก็เริ่มค้นหาจากตำแหน่ง `2`:

```js run
let str = 'Widget with id';

alert( str.indexOf('id', 2) ) // 12
```

ถ้าต้องการหาทุกตำแหน่งที่ปรากฏ ก็ใช้ `indexOf` ในลูปได้ โดยทุกรอบจะเริ่มค้นจากตำแหน่งถัดจากที่พบล่าสุด:

```js run
let str = 'As sly as a fox, as strong as an ox';

let target = 'as'; // หา "as" ในสตริง

let pos = 0;
while (true) {
  let foundPos = str.indexOf(target, pos);
  if (foundPos == -1) break;

  alert( `Found at ${foundPos}` );
  pos = foundPos + 1; // ค้นหาต่อจากตำแหน่งถัดไป
}
```

อัลกอริทึมเดียวกันเขียนให้สั้นลงได้:

```js run
let str = "As sly as a fox, as strong as an ox";
let target = "as";

*!*
let pos = -1;
while ((pos = str.indexOf(target, pos + 1)) != -1) {
  alert( pos );
}
*/!*
```

```smart header="`str.lastIndexOf(substr, position)`"
มีเมธอดคล้ายกันคือ [str.lastIndexOf(substr, position)](mdn:js/String/lastIndexOf) ที่ค้นหาจากท้ายสตริงไปยังจุดเริ่มต้น

จะแสดงตำแหน่งที่พบในลำดับย้อนกลับ
```

มีข้อควรระวังเล็กน้อยเมื่อใช้ `indexOf` ใน `if` — ใช้แบบนี้ตรงๆ ไม่ได้:

```js run
let str = "Widget with id";

if (str.indexOf("Widget")) {
    alert("We found it"); // ไม่ทำงานอย่างที่คาด!
}
```

`alert` ด้านบนไม่แสดงผลเพราะ `str.indexOf("Widget")` คืนค่า `0` (หมายความว่าพบที่ตำแหน่งเริ่มต้น) แต่ `if` ถือว่า `0` เป็น `false`

ดังนั้นต้องเช็คกับ `-1` แบบนี้:

```js run
let str = "Widget with id";

*!*
if (str.indexOf("Widget") != -1) {
*/!*
    alert("We found it"); // ตอนนี้ทำงานแล้ว!
}
```

### includes, startsWith, endsWith

เมธอดที่ใหม่กว่าคือ [str.includes(substr, pos)](mdn:js/String/includes) ซึ่งคืนค่า `true/false` ตามว่า `str` มี `substr` อยู่หรือไม่

เหมาะสำหรับกรณีที่ต้องการแค่เช็คว่ามีหรือไม่ โดยไม่ต้องการตำแหน่ง:

```js run
alert( "Widget with id".includes("Widget") ); // true

alert( "Hello".includes("Bye") ); // false
```

อาร์กิวเมนต์ตัวที่สองของ `str.includes` เป็นตัวเลือก กำหนดตำแหน่งเริ่มค้นหา:

```js run
alert( "Widget".includes("id") ); // true
alert( "Widget".includes("id", 3) ); // false, ตั้งแต่ตำแหน่งที่ 3 ไม่มี "id"
```

เมธอด [str.startsWith](mdn:js/String/startsWith) และ [str.endsWith](mdn:js/String/endsWith) ทำตามชื่อเลย — เช็คว่าสตริงขึ้นต้นหรือลงท้ายด้วยข้อความที่กำหนดหรือไม่:

```js run
alert( "*!*Wid*/!*get".startsWith("Wid") ); // true ขึ้นต้นด้วย "Wid"
alert( "Wid*!*get*/!*".endsWith("get") ); // true ลงท้ายด้วย "get"
```

## การดึงสตริงย่อย

JavaScript มี 3 เมธอดสำหรับดึงสตริงย่อย: `substring`, `substr` และ `slice`

`str.slice(start [, end])`
: คืนค่าส่วนของสตริงจาก `start` ถึง (แต่ไม่รวม) `end`

    เช่น:

    ```js run
    let str = "stringify";
    alert( str.slice(0, 5) ); // 'strin', สตริงย่อยจาก 0 ถึง 5 (ไม่รวม 5)
    alert( str.slice(0, 1) ); // 's', จาก 0 ถึง 1 แต่ไม่รวม 1 ดังนั้นเป็นเพียงอักขระที่ 0
    ```

    ถ้าไม่ระบุอาร์กิวเมนต์ตัวที่สอง `slice` จะดึงจนถึงท้ายสตริง:

    ```js run
    let str = "st*!*ringify*/!*";
    alert( str.slice(2) ); // 'ringify', จากตำแหน่งที่ 2 จนถึงท้าย
    ```

    ใส่ค่าลบสำหรับ `start/end` ได้ด้วย หมายถึงนับตำแหน่งจากท้ายสตริง:

    ```js run
    let str = "strin*!*gif*/!*y";

    // เริ่มที่ตำแหน่งที่ 4 จากขวา จบที่ตำแหน่งที่ 1 จากขวา
    alert( str.slice(-4, -1) ); // 'gif'
    ```

`str.substring(start [, end])`
: คืนค่าส่วนของสตริง*ระหว่าง* `start` และ `end` (ไม่รวม `end`)

    คล้ายกับ `slice` แต่ต่างตรงที่ `substring` ยอมให้ `start` มากกว่า `end` ได้ (จะสลับค่าให้อัตโนมัติ)

    เช่น:

    ```js run
    let str = "st*!*ring*/!*ify";

    // ทั้งสองคำสั่งนี้ให้ผลลัพธ์เหมือนกันสำหรับ substring
    alert( str.substring(2, 6) ); // "ring"
    alert( str.substring(6, 2) ); // "ring"

    // แต่ให้ผลต่างกันสำหรับ slice:
    alert( str.slice(2, 6) ); // "ring" (เหมือนกัน)
    alert( str.slice(6, 2) ); // "" (สตริงว่าง)

    ```

    ค่าลบ `substring` จะถือเป็น `0` ซึ่งต่างจาก `slice`

`str.substr(start [, length])`
: คืนค่าส่วนของสตริงจาก `start` โดยกำหนด `length` (ความยาว)

    ต่างจากสองเมธอดก่อนหน้าตรงที่ระบุ `length` แทนตำแหน่งสิ้นสุด:

    ```js run
    let str = "st*!*ring*/!*ify";
    alert( str.substr(2, 4) ); // 'ring', จากตำแหน่งที่ 2 ดึง 4 อักขระ
    ```

    อาร์กิวเมนต์แรกเป็นค่าลบได้ เพื่อนับจากท้ายสตริง:

    ```js run
    let str = "strin*!*gi*/!*fy";
    alert( str.substr(-4, 2) ); // 'gi', จากตำแหน่งที่ 4 จากท้าย ดึง 2 อักขระ
    ```

    เมธอดนี้อยู่ใน [Annex B](https://tc39.es/ecma262/#sec-string.prototype.substr) ของสเปคภาษา หมายความว่ามีเฉพาะใน JavaScript engine ของเบราว์เซอร์เท่านั้น และไม่แนะนำให้ใช้ แต่ในทางปฏิบัติใช้ได้ทุกที่

| เมธอด | เลือก... | ค่าลบ |
|--------|-----------|-----------|
| `slice(start, end)` | จาก `start` ถึง `end` (ไม่รวม `end`) | อนุญาตค่าลบ |
| `substring(start, end)` | ระหว่าง `start` และ `end` (ไม่รวม `end`) | ค่าลบหมายถึง `0` |
| `substr(start, length)` | จาก `start` ดึง `length` อักขระ | อนุญาต `start` เป็นลบ |

```smart header="ควรเลือกใช้ตัวไหนดี?"
ทั้งสามเมธอดทำงานได้หมด ว่ากันตามจริง `substr` มีข้อเสียตรงที่ไม่ได้อยู่ในสเปคหลัก แต่อยู่ใน Annex B ซึ่งครอบคลุมฟีเจอร์สำหรับเบราว์เซอร์เป็นหลัก สภาพแวดล้อมอื่นอาจไม่รองรับ

ในสองตัวที่เหลือ `slice` ยืดหยุ่นกว่า รองรับค่าลบได้ และเขียนสั้นกว่า

สำหรับการใช้งานจริง จำแค่ `slice` ก็เพียงพอแล้ว
```

## การเปรียบเทียบสตริง

อย่างที่เรารู้จากบท <info:comparison> สตริงจะถูกเปรียบเทียบทีละอักขระตามลำดับตัวอักษร

แต่ก็มีเรื่องแปลกๆ อยู่บ้าง

1. ตัวพิมพ์เล็กจะ "มากกว่า" ตัวพิมพ์ใหญ่เสมอ:

    ```js run
    alert( 'a' > 'Z' ); // true
    ```

2. ตัวอักษรที่มีเครื่องหมายกำกับ (diacritical marks) จะเรียงแบบ "ไม่ปกติ":

    ```js run
    alert( 'Österreich' > 'Zealand' ); // true
    ```

    ผลลัพธ์นี้อาจดูแปลก ถ้าเราเรียงชื่อประเทศ คนทั่วไปจะคาดว่า `Zealand` ควรมาหลัง `Österreich`

เพื่อเข้าใจว่าทำไมถึงเป็นแบบนี้ ต้องรู้ว่าสตริงใน JavaScript ถูกเข้ารหัสด้วย [UTF-16](https://en.wikipedia.org/wiki/UTF-16) — แต่ละอักขระจะมีรหัสตัวเลขที่สอดคล้องกัน

มีเมธอดพิเศษสำหรับดึงรหัสของอักขระและสร้างอักขระจากรหัส:

`str.codePointAt(pos)`
: คืนค่ารหัสตัวเลข (decimal) ของอักขระที่ตำแหน่ง `pos`:

    ```js run
    // ตัวอักษรที่มีตัวพิมพ์ต่างกันจะมีรหัสต่างกัน
    alert( "Z".codePointAt(0) ); // 90
    alert( "z".codePointAt(0) ); // 122
    alert( "z".codePointAt(0).toString(16) ); // 7a (ถ้าเราต้องการค่าฐานสิบหก)
    ```

`String.fromCodePoint(code)`
: สร้างอักขระจากรหัส Unicode ที่กำหนด:

    ```js run
    alert( String.fromCodePoint(90) ); // Z
    alert( String.fromCodePoint(0x5a) ); // Z (เราสามารถใช้ค่าฐานสิบหกเป็นอาร์กิวเมนต์ได้ด้วย)
    ```

เราสามารถทดลองดูอักขระที่มีรหัสระหว่าง 65 ถึง 220 (ซึ่งครอบคลุมตัวอักษรละตินและสัญลักษณ์พิเศษบางตัว) ได้ดังนี้:

```js run
let str = '';

for (let i = 65; i <= 220; i++) {
  str += String.fromCodePoint(i);
}
alert( str );
// ผลลัพธ์:
// ABCDEFGHIJKLMNOPQRSTUVWXYZ[\]^_`abcdefghijklmnopqrstuvwxyz{|}~
// ¡¢£¤¥¦§¨©ª«¬­®¯°±²³´µ¶·¸¹º»¼½¾¿ÀÁÂÃÄÅÆÇÈÉÊËÌÍÎÏÐÑÒÓÔÕÖ×ØÙÚÛÜ
```

จากผลลัพธ์นี้ เราจะเห็นว่าตัวอักษรตัวพิมพ์ใหญ่มาก่อน ตามด้วยสัญลักษณ์พิเศษบางตัว แล้วจึงเป็นตัวอักษรตัวพิมพ์เล็ก และตัวอักษรที่มีเครื่องหมายกำกับ (เช่น Ö) อยู่ท้ายสุด

ตอนนี้เราเข้าใจแล้วว่าทำไม `'a' > 'Z'`:

1. อักขระถูกเปรียบเทียบโดยใช้รหัสตัวเลขของพวกมัน
2. รหัสที่มากกว่าหมายถึงอักขระนั้น "มากกว่า" ในการเปรียบเทียบ
3. รหัสสำหรับ `a` (97) มากกว่ารหัสสำหรับ `Z` (90)

### การเปรียบเทียบที่ถูกต้องตามภาษา

อัลกอริทึมการเปรียบเทียบสตริง "ที่ถูกต้อง" ตามหลักภาษาศาสตร์นั้นซับซ้อนกว่าที่คิด เพราะภาษาต่างๆ มีกฎการเรียงลำดับตัวอักษรที่แตกต่างกัน

โชคดีที่เบราว์เซอร์สมัยใหม่ (และ Node.js) สนับสนุนมาตรฐานการทำให้เป็นสากล [ECMA-402](https://www.ecma-international.org/publications-and-standards/standards/ecma-402/) ซึ่งให้วิธีการเปรียบเทียบสตริงที่คำนึงถึงกฎของแต่ละภาษา

เราสามารถใช้เมธอด [str.localeCompare(str2)](mdn:js/String/localeCompare) เพื่อเปรียบเทียบสตริงตามกฎภาษา:

ตัวอย่างเช่น:

```js run
alert( 'Österreich'.localeCompare('Zealand') ); // -1
```
เข้าใจครับ ผมจะขยายความเฉพาะส่วนที่เกี่ยวกับ `localeCompare` และสรุปตามที่ระบุไว้ในต้นฉบับ โดยไม่เพิ่มเติมเนื้อหาใหม่

เมธอด `localeCompare` มีอาร์กิวเมนต์เพิ่มเติมที่ช่วยให้เราสามารถปรับแต่งการเปรียบเทียบได้ดังนี้:

```js
str.localeCompare(str2, [locales, [options]])
```

- `locales`: ช่วยระบุภาษาที่ใช้ในการเปรียบเทียบ โดยค่าเริ่มต้นจะใช้ภาษาตามสภาพแวดล้อมของระบบ
- `options`: เป็นวัตถุที่ใช้กำหนดกฎเพิ่มเติมในการเปรียบเทียบ เช่น การคำนึงถึงตัวพิมพ์ใหญ่-เล็ก หรือการปฏิบัติต่ออักขระที่มีเครื่องหมายกำกับ

การใช้อาร์กิวเมนต์เหล่านี้ช่วยให้เราสามารถปรับแต่งการเปรียบเทียบให้เหมาะกับความต้องการเฉพาะได้ เช่น การเปรียบเทียบโดยไม่คำนึงถึงตัวพิมพ์ใหญ่-เล็ก หรือการปฏิบัติต่อตัวอักษรที่มีเครื่องหมายกำกับ (เช่น "a" และ "á") ให้เหมือนกัน

## สรุป

เรามาสรุปสาระสำคัญเกี่ยวกับการจัดการสตริงใน JavaScript กัน:

1. **การสร้างสตริง**: สามารถใช้เครื่องหมายคำพูดเดี่ยว (''), คู่ ("") หรือ backticks (``). Backticks มีความสามารถพิเศษในการแทรกนิพจน์และสร้างสตริงหลายบรรทัด

2. **การเข้าถึงอักขระ**: ใช้วงเล็บเหลี่ยม `[]` หรือเมธอด `at()` โดย `at()` สามารถใช้ดัชนีเชิงลบได้

3. **การเปลี่ยนตัวพิมพ์**: ใช้ `toLowerCase()` และ `toUpperCase()`

4. **การค้นหาสตริงย่อย**: 
   - `indexOf()`: หาตำแหน่งแรกที่พบ
   - `includes()`: ตรวจสอบการมีอยู่
   - `startsWith()` และ `endsWith()`: ตรวจสอบการขึ้นต้นและลงท้าย

5. **การดึงสตริงย่อย**: 
   - `slice(start, end)`: ดึงจาก start ถึง (ไม่รวม) end, รองรับดัชนีลบ
   - `substring(start, end)`: คล้าย slice แต่ไม่รองรับดัชนีลบ
   - `substr(start, length)`: ดึงตามความยาวที่กำหนด (ไม่แนะนำให้ใช้ในโค้ดใหม่)

6. **การเปรียบเทียบสตริง**: 
   - ใช้เครื่องหมาย `<`, `>`, `==`, `===` สำหรับการเปรียบเทียบทั่วไป
   - ใช้ `localeCompare()` สำหรับการเปรียบเทียบที่คำนึงถึงภาษา

7. **อักขระพิเศษ**: เช่น `\n` สำหรับขึ้นบรรทัดใหม่, `\t` สำหรับแท็บ, `\'` และ `\"` สำหรับเครื่องหมายคำพูด

8. **ความยาวของสตริง**: ใช้คุณสมบัติ `length`

9. **สตริงเป็นข้อมูลที่ไม่สามารถเปลี่ยนแปลงได้ (immutable)**: การแก้ไขสตริงต้องสร้างสตริงใหม่เสมอ

10. **การทำงานกับ Unicode**: ใช้ `codePointAt()` และ `String.fromCodePoint()` สำหรับการจัดการรหัส Unicode

เมื่อทำงานกับสตริง สิ่งสำคัญคือต้องเลือกใช้เมธอดที่เหมาะสมกับงาน และระมัดระวังเรื่องประสิทธิภาพเมื่อต้องทำงานกับสตริงขนาดใหญ่หรือการดำเนินการซ้ำๆ

นอกจากนี้ JavaScript ยังมีเมธอดอื่นๆ สำหรับการจัดการสตริงที่มีประโยชน์ เช่น:

- `trim()`: ลบช่องว่างจากจุดเริ่มต้นและจุดสิ้นสุดของสตริง
- `repeat(n)`: ทำซ้ำสตริง `n` ครั้ง
- `padStart()` และ `padEnd()`: เติมสตริงให้ได้ความยาวที่กำหนด

สำหรับการค้นหาและแทนที่ที่ซับซ้อนมากขึ้น JavaScript รองรับการใช้นิพจน์ทั่วไป (Regular Expressions) ซึ่งเป็นเครื่องมือที่ทรงพลังสำหรับการจัดการกับรูปแบบของสตริง

สุดท้าย เมื่อทำงานกับสตริงที่มีอักขระพิเศษหรือต้องการความถูกต้องในการจัดการกับ Unicode ควรศึกษาเพิ่มเติมเกี่ยวกับการทำงานของ Unicode ใน JavaScript เพื่อหลีกเลี่ยงปัญหาที่อาจเกิดขึ้นกับอักขระบางตัว ดูเพิ่มเติมได้ที่บทเรียนนี้ info:unicode
