# ตัวเลข

ใน JavaScript ยุคใหม่ มีตัวเลขอยู่สองประเภท:

1. ตัวเลขทั่วไปใน JavaScript เก็บในรูปแบบ 64 บิต [IEEE-754](https://en.wikipedia.org/wiki/IEEE_754) หรือที่เรียกว่า "double precision floating point numbers" เป็นตัวเลขที่เราใช้กันบ่อยที่สุด และจะพูดถึงในบทนี้

2. BigInt ใช้แทนจำนวนเต็มที่มีความยาวไม่จำกัด บางครั้งจำเป็นต้องใช้ เพราะตัวเลขทั่วไปมีค่าได้ไม่เกิน <code>(2<sup>53</sup>-1)</code> หรือไม่น้อยกว่า <code>-(2<sup>53</sup>-1)</code> ดังที่เคยกล่าวไว้ในบท <info:types> เนื่องจาก BigInt ใช้ในบางกรณีพิเศษเท่านั้น จึงแยกไว้ในบทเฉพาะ <info:bigint>

ในบทนี้เราจะพูดถึงตัวเลขทั่วไป มาเรียนรู้เพิ่มเติมกัน

## วิธีเขียนตัวเลขแบบอื่นๆ

สมมติว่าเราต้องเขียนหนึ่งพันล้าน วิธีที่ชัดเจนคือ:

```js
let billion = 1000000000;
```

นอกจากนี้ยังใช้เครื่องหมายขีดล่าง `_` เป็นตัวคั่นได้:

```js
let billion = 1_000_000_000;
```

ขีดล่าง `_` ตรงนี้ทำหน้าที่เป็น "[น้ำตาลทางไวยากรณ์ (syntactic sugar)](https://en.wikipedia.org/wiki/Syntactic_sugar)" ช่วยให้ตัวเลขอ่านง่ายขึ้น JavaScript engine จะมองข้าม `_` ระหว่างตัวเลข ดังนั้นจึงเป็นหนึ่งพันล้านเหมือนกันกับข้างบน

แต่ในชีวิตจริง เราขี้เกียจเขียนศูนย์ต่อกันยาวๆ มักเขียนแบบย่อ เช่น `"1bn"` สำหรับหนึ่งพันล้าน หรือ `"7.3bn"` สำหรับเจ็ดพันสามร้อยล้าน ตัวเลขใหญ่ส่วนมากก็เขียนแบบนี้

ใน JavaScript เราย่อตัวเลขได้โดยเพิ่มตัวอักษร `"e"` ต่อท้ายแล้วระบุจำนวนศูนย์:

```js run
let billion = 1e9;  // 1 พันล้าน คือ 1 ตามด้วยศูนย์ 9 ตัว

alert( 7.3e9 );  // 7.3 พันล้าน (เท่ากับ 7300000000 หรือ 7_300_000_000)
```

พูดง่ายๆ คือ `e` คูณตัวเลขด้วย `1` ตามด้วยจำนวนศูนย์ที่ระบุ

```js
1e3 === 1 * 1000; // e3 หมายถึง *1000
1.23e6 === 1.23 * 1000000; // e6 หมายถึง *1000000
```

ทีนี้มาเขียนตัวเลขที่เล็กมากๆ กัน สมมติว่า 1 ไมโครวินาที (หนึ่งในล้านของวินาที):

```js
let mсs = 0.000001;
```

เช่นเดียวกับก่อนหน้านี้ การใช้ `"e"` ช่วยได้ ถ้าเราไม่อยากเขียนศูนย์เยอะๆ เราก็เขียนแบบนี้ได้:

```js
let mcs = 1e-6; // ศูนย์หกตัวทางซ้ายของ 1
```

ถ้านับจำนวนศูนย์ใน `0.000001` จะมี 6 ตัว ดังนั้นจึงเป็น `1e-6`

กล่าวอีกอย่างคือ ตัวเลขลบหลัง `"e"` หมายถึงการหารด้วย 1 ตามด้วยจำนวนศูนย์ที่ระบุ:

```js
// -3 หมายถึงหารด้วย 1000 (3 ศูนย์)
1e-3 === 1 / 1000; // 0.001

// -6 หมายถึงหารด้วย 1,000,000 (6 ศูนย์)
1.23e-6 === 1.23 / 1000000; // 0.00000123

// ตัวอย่างกับตัวเลขที่ใหญ่ขึ้น
1234e-2 === 1234 / 100; // 12.34, จุดทศนิยมขยับไป 2 ตำแหน่ง
```

### เลขฐานสิบหก ฐานสอง และฐานแปด

[เลขฐานสิบหก](https://en.wikipedia.org/wiki/Hexadecimal) ใช้กันมากในจาวาสคริปต์ เพื่อแสดงสี เข้ารหัสตัวอักษร และอื่นๆ อีกมาก จึงมีวิธีเขียนแบบย่อ คือใช้ `0x` นำหน้าตัวเลข

ตัวอย่าง:

```js run
alert( 0xff ); // 255
alert( 0xFF ); // 255 (ไม่ต่างกัน ตัวพิมพ์เล็กหรือใหญ่ไม่สำคัญ)
```

ระบบเลขฐานสองและฐานแปดใช้น้อย แต่ก็รองรับ โดยใช้คำนำหน้า `0b` และ `0o`:


```js run
let a = 0b11111111; // รูปแบบเลขฐานสองของ 255
let b = 0o377; // รูปแบบเลขฐานแปดของ 255

alert( a == b ); // จริง เป็นเลข 255 เหมือนกันทั้งสองฝั่ง
```

มีแค่ 3 ระบบเลขที่รองรับแบบนี้ สำหรับระบบเลขอื่นๆ เราควรใช้ฟังก์ชัน `parseInt` (ซึ่งเราจะเห็นในภายหลังในบทนี้)

## toString(base)

เมธอด `num.toString(base)` คืนค่าสตริงที่แสดง `num` ในระบบเลขฐาน `base`

ตัวอย่าง:
```js run
let num = 255;

alert( num.toString(16) );  // ff
alert( num.toString(2) );   // 11111111
```

`base` อาจมีค่าตั้งแต่ `2` ถึง `36` โดยค่าเริ่มต้นคือ `10`

กรณีใช้งานทั่วไปคือ:

- **base=16** ใช้สำหรับสีในรูปแบบเลขฐานสิบหก, เข้ารหัสตัวอักษร ฯลฯ ตัวเลขอาจเป็น `0..9` หรือ `A..F`
- **base=2** ส่วนใหญ่ใช้สำหรับดีบัก bitwise operations ตัวเลขจะเป็น `0` หรือ `1`
- **base=36** เป็นค่าสูงสุด ตัวเลขจะเป็น `0..9` หรือ `A..Z` โดยใช้ตัวอักษรละตินทั้งหมดแทนตัวเลข กรณีที่สนุกแต่มีประโยชน์ของฐาน `36` คือเมื่อต้องการย่อตัวระบุตัวเลขที่ยาวให้สั้นลง เช่น ทำ URL สั้นๆ — แค่แสดงในระบบเลขฐาน `36`:

    ```js run
    alert( 123456..toString(36) ); // 2n9c
    ```

```warn header="จุดสองจุดเพื่อเรียกเมธอด"
สังเกตนะว่าจุดสองจุดใน `123456..toString(36)` ไม่ใช่พิมพ์ผิด — ถ้าต้องการเรียกเมธอดกับตัวเลขโดยตรง เช่น `toString` ในตัวอย่างข้างต้น ต้องใส่จุดสองจุด `..` หลังตัวเลข

ถ้าใส่จุดเดียว: `123456.toString(36)` จะเกิด error เพราะ JavaScript คิดว่าจุดแรกคือจุดทศนิยม พอใส่จุดอีกจุด JavaScript จะรู้ว่าส่วนทศนิยมว่างเปล่าและจะเรียกเมธอดแทน

หรือจะเขียน `(123456).toString(36)` ก็ได้เช่นกัน

## การปัดเศษ

หนึ่งในสิ่งที่ทำบ่อยที่สุดกับตัวเลขคือการปัดเศษ

มีฟังก์ชันในตัวหลายตัวสำหรับปัดเศษ:

`Math.floor`
: ปัดลง: `3.1` เป็น `3`, `-1.1` เป็น `-2`

`Math.ceil`
: ปัดขึ้น: `3.1` เป็น `4`, `-1.1` เป็น `-1`

`Math.round`
: ปัดเศษไปยังจำนวนเต็มที่ใกล้ที่สุด: `3.1` เป็น `3`, `3.6` เป็น `4` กรณีกึ่งกลาง `3.5` ปัดขึ้นเป็น `4` และ `-3.5` ปัดขึ้นเป็น `-3`

`Math.trunc` (ไม่รองรับใน Internet Explorer)
: ตัดส่วนทศนิยมทิ้งโดยไม่ปัดเศษ: `3.1` เป็น `3`, `-1.1` เป็น `-1`

ตารางสรุปความแตกต่างระหว่างฟังก์ชันเหล่านี้:

|   | `Math.floor` | `Math.ceil` | `Math.round` | `Math.trunc` |
|---|---------|--------|---------|---------|
|`3.1`|  `3`    |   `4`  |    `3`  |   `3`   |
|`3.5`|  `3`    |   `4`  |    `4`  |   `3`   |
|`3.6`|  `3`    |   `4`  |    `4`  |   `3`   |
|`-1.1`|  `-2`    |   `-1`  |    `-1`  |   `-1`   |
|`-1.5`|  `-2`    |   `-1`  |    `-1`  |   `-1`   |
|`-1.6`|  `-2`    |   `-1`  |    `-2`  |   `-1`   |


ฟังก์ชันเหล่านี้ครอบคลุมการจัดการส่วนทศนิยมทั้งหมด แล้วถ้าต้องการปัดเศษไปยังตำแหน่งทศนิยมที่ `n` ล่ะ?

เช่น เรามี `1.2345` และต้องการปัดเศษให้เหลือ 2 ตำแหน่งทศนิยม เป็น `1.23`

ทำได้สองวิธี:

1. คูณแล้วหาร

    เช่น ถ้าต้องการปัดเศษให้เหลือ 2 ตำแหน่งทศนิยม ก็คูณด้วย `100` เรียกฟังก์ชันปัดเศษ แล้วหารกลับ

    ```js run
    let num = 1.23456;

    alert( Math.round(num * 100) / 100 ); // 1.23456 -> 123.456 -> 123 -> 1.23
    ```

2. เมธอด [toFixed(n)](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Number/toFixed) ปัดเศษไปยัง `n` ตำแหน่งหลังจุดทศนิยม แล้วคืนค่าเป็นสตริง

    ```js run
    let num = 12.34;
    alert( num.toFixed(1) ); // "12.3"
    ```

    ปัดขึ้นหรือลงไปยังค่าที่ใกล้ที่สุด คล้ายกับ `Math.round`:

    ```js run
    let num = 12.36;
    alert( num.toFixed(1) ); // "12.4"
    ```

    สังเกตด้วยว่าผลลัพธ์ของ `toFixed` เป็นสตริง ถ้าส่วนทศนิยมสั้นกว่าที่ต้องการ จะเพิ่มศูนย์ต่อท้าย:

    ```js run
    let num = 12.34;
    alert( num.toFixed(5) ); // "12.34000", เพิ่มศูนย์เพื่อให้ครบ 5 หลัก
    ```

    แปลงกลับเป็นตัวเลขได้โดยใช้เครื่องหมายบวกหรือเรียก `Number()` เช่น `+num.toFixed(5)`

## การคำนวณที่ไม่แม่นยำ

ภายใน ตัวเลขเก็บในรูปแบบ 64 บิต [IEEE-754](https://en.wikipedia.org/wiki/IEEE_754) โดยใช้ 52 บิตเก็บตัวเลข 11 บิตเก็บตำแหน่งจุดทศนิยม และ 1 บิตสำหรับเครื่องหมาย

ถ้าตัวเลขใหญ่เกินไป จะล้นพื้นที่ 64 บิต กลายเป็นค่า `Infinity`:

```js run
alert( 1e500 ); // Infinity
```

สิ่งที่อาจไม่ชัดเจนนัก แต่เกิดขึ้นบ่อยคือ การสูญเสียความแม่นยำ

ลองดูการเปรียบเทียบนี้ (ผลลัพธ์เป็นเท็จ!):

```js run
alert( 0.1 + 0.2 == 0.3 ); // *!*false*/!*
```

ใช่แล้ว! ถ้าเช็คว่าผลรวมของ `0.1` กับ `0.2` เท่ากับ `0.3` ไหม — คำตอบคือ `false`

แปลกดีใช่ไหม? แล้วถ้าไม่ใช่ `0.3` มันได้เท่าไหร่ล่ะ?

```js run
alert( 0.1 + 0.2 ); // 0.30000000000000004
```

โอ้! ลองนึกภาพว่ากำลังทำเว็บขายของออนไลน์ แล้วลูกค้าใส่สินค้ามูลค่า `$0.10` กับ `$0.20` ลงในตะกร้า ยอดรวมจะกลายเป็น `$0.30000000000000004` ใครเห็นก็ต้องตกใจ

แต่ทำไมถึงเป็นแบบนี้?

ตัวเลขเก็บในหน่วยความจำเป็นไบนารี — ลำดับของ 0 กับ 1 แต่เศษส่วนอย่าง `0.1`, `0.2` ที่ดูง่ายในระบบฐานสิบ กลับเป็นเศษส่วนไม่รู้จบในระบบไบนารี

```js run
alert(0.1.toString(2)); // 0.0001100110011001100110011001100110011001100110011001101
alert(0.2.toString(2)); // 0.001100110011001100110011001100110011001100110011001101
alert((0.1 + 0.2).toString(2)); // 0.0100110011001100110011001100110011001100110011001101
```

`0.1` คืออะไร? คือหนึ่งหารสิบ (`1/10`) นั่นเอง ในระบบฐานสิบ ตัวเลขแบบนี้แสดงได้ง่าย ลองเปรียบเทียบกับหนึ่งหารสาม (`1/3`) ที่กลายเป็นเศษส่วนไม่รู้จบ `0.33333(3)`

การหารด้วยกำลังของ 10 ทำงานได้ดีในระบบฐานสิบ แต่หารด้วย 3 ไม่ได้ ด้วยเหตุผลเดียวกัน ในระบบไบนารี การหารด้วยกำลังของ 2 ทำงานได้ดี แต่ `1/10` กลับกลายเป็นเศษส่วนไบนารีไม่รู้จบ

ไม่มีทางเก็บ *0.1 หรือ 0.2 ได้แม่นยำ* ในระบบไบนารี เช่นเดียวกับที่ไม่มีทางเขียน 1/3 เป็นเศษส่วนทศนิยมได้ตรงเป๊ะ

รูปแบบตัวเลข IEEE-754 แก้ปัญหานี้โดยปัดเศษไปยังตัวเลขที่ใกล้ที่สุด กฎการปัดเศษเหล่านี้ทำให้เราไม่เห็น "ความคลาดเคลื่อนเล็กน้อย" แต่จริงๆ แล้วมีอยู่

ลองดู:
```js run
alert( 0.1.toFixed(20) ); // 0.10000000000000000555
```

พอบวกสองตัวเลขเข้าด้วยกัน "ความคลาดเคลื่อน" ก็รวมกันด้วย

นั่นคือเหตุผลที่ `0.1 + 0.2` ไม่เท่ากับ `0.3` อย่างแม่นยำ

```smart header="ไม่ใช่แค่ JavaScript"
ปัญหาเดียวกันนี้มีอยู่ในภาษาโปรแกรมอื่นๆ อีกหลายภาษา

PHP, Java, C, Perl, Ruby ให้ผลลัพธ์เหมือนกัน เพราะใช้รูปแบบตัวเลขเดียวกัน
```

แก้ปัญหานี้ได้ไหม? ได้แน่นอน วิธีที่น่าเชื่อถือที่สุดคือปัดเศษผลลัพธ์ด้วยเมธอด [toFixed(n)](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Number/toFixed):

```js run
let sum = 0.1 + 0.2;
alert( sum.toFixed(2) ); // "0.30"
```

สังเกตว่า `toFixed` คืนค่าเป็นสตริงเสมอ และรับประกันว่าจะมี 2 ตำแหน่งหลังจุดทศนิยม ซึ่งสะดวกถ้าทำร้านค้าออนไลน์แล้วต้องแสดง `$0.30` สำหรับกรณีอื่นๆ ใช้เครื่องหมายบวกแปลงกลับเป็นตัวเลขได้:

```js run
let sum = 0.1 + 0.2;
alert( +sum.toFixed(2) ); // 0.3
```

อีกวิธีคือคูณตัวเลขด้วย 100 (หรือมากกว่า) ชั่วคราว เพื่อเปลี่ยนเป็นจำนวนเต็ม คำนวณแล้วค่อยหารกลับ วิธีนี้ช่วยลดความคลาดเคลื่อนได้บ้าง แต่ยังไม่หมดไปเวลาหาร:

```js run
alert( (0.1 * 10 + 0.2 * 10) / 10 ); // 0.3
alert( (0.28 * 100 + 0.14 * 100) / 100); // 0.4200000000000001
```

ดังนั้น วิธีคูณ/หารช่วยลดความคลาดเคลื่อนได้ แต่ไม่ได้หมดไปทั้งหมด

บางทีอาจคิดจะหลีกเลี่ยงเศษส่วนทั้งหมด เช่น ถ้าทำร้านค้า อาจเก็บราคาเป็นเซนต์แทนดอลลาร์ แต่ถ้าลดราคา 30% ล่ะ? ในทางปฏิบัติ การหลีกเลี่ยงเศษส่วนทั้งหมดแทบเป็นไปไม่ได้ — แค่ปัดเศษตัดทศนิยมเมื่อจำเป็นก็พอ

ลองรันโค้ดนี้ดู:

```js run
// สวัสดี! ฉันเป็นตัวเลขที่เพิ่มขึ้นเอง!
alert( 9999999999999999 ); // แสดง 10000000000000000
```

นี่เกิดจากปัญหาเดียวกัน — สูญเสียความแม่นยำ มี 64 บิตสำหรับตัวเลข แต่ 52 บิตที่ใช้เก็บตัวเลขไม่พอ ตัวเลขที่มีนัยสำคัญน้อยที่สุดจึงหายไป

JavaScript ไม่ฟ้อง error ในกรณีแบบนี้ แค่พยายามจัดตัวเลขให้พอดีกับรูปแบบที่มี แต่น่าเสียดายที่รูปแบบนี้ไม่ใหญ่พอ
````

```smart header="สองศูนย์"
อีกเรื่องแปลกจากการเก็บตัวเลขภายในคือ มีศูนย์สองแบบ: `0` กับ `-0`

เพราะเครื่องหมายเก็บด้วยบิตเดียว จะตั้งค่าหรือไม่ตั้งก็ได้สำหรับตัวเลขใดๆ รวมถึงศูนย์

ส่วนใหญ่ความแตกต่างนี้ไม่มีผลอะไร เพราะตัวดำเนินการถือว่าทั้งสองค่าเหมือนกัน
```

## การทดสอบ: isFinite และ isNaN

จำค่าตัวเลขพิเศษสองตัวนี้ได้ไหม?

- `Infinity` (และ `-Infinity`) เป็นค่าตัวเลขพิเศษที่มากกว่า (หรือน้อยกว่า) ทุกอย่าง
- `NaN` แทน error

ค่าเหล่านี้มีชนิดเป็น `number` แต่ไม่ใช่ตัวเลข "ปกติ" จึงมีฟังก์ชันพิเศษสำหรับตรวจสอบ:

- `isNaN(value)` แปลงอาร์กิวเมนต์เป็นตัวเลขแล้วทดสอบว่าเป็น `NaN`:

    ```js run
    alert( isNaN(NaN) ); // จริง
    alert( isNaN("str") ); // จริง
    ```

    แต่จำเป็นต้องใช้ฟังก์ชันนี้ไหม? แค่เช็ค `=== NaN` ไม่ได้หรอ? คำตอบคือไม่ได้ เพราะ `NaN` เป็นค่าพิเศษที่ไม่เท่ากับอะไรเลย — แม้กระทั่งตัวเอง:

    ```js run
    alert( NaN === NaN ); // เท็จ
    ```

- `isFinite(value)` แปลงอาร์กิวเมนต์เป็นตัวเลข แล้วคืนค่า `true` ถ้าเป็นตัวเลขปกติ ไม่ใช่ `NaN/Infinity/-Infinity`:

    ```js run
    alert( isFinite("15") ); // จริง
    alert( isFinite("str") ); // เท็จ เพราะเป็นค่าพิเศษ: NaN
    alert( isFinite(Infinity) ); // เท็จ เพราะเป็นค่าพิเศษ: Infinity
    ```

บางครั้ง `isFinite` ใช้ตรวจสอบว่าค่าสตริงเป็นตัวเลขปกติหรือไม่:


```js run
let num = +prompt("ป้อนตัวเลข", '');

// จะเป็น true ยกเว้นป้อน Infinity, -Infinity หรือค่าที่ไม่ใช่ตัวเลข
alert( isFinite(num) );
```

สังเกตด้วยว่าสตริงว่างเปล่าหรือสตริงที่มีแต่ช่องว่าง จะถูกมองเป็น `0` ในทุกฟังก์ชันตัวเลข รวมถึง `isFinite`

````smart header="`Number.isNaN` และ `Number.isFinite`"
เมธอด [Number.isNaN](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Number/isNaN) และ [Number.isFinite](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Number/isFinite) เป็นเวอร์ชัน "เข้มงวด" ของ `isNaN` กับ `isFinite` โดยไม่แปลงอาร์กิวเมนต์เป็นตัวเลขอัตโนมัติ แต่เช็คว่าอาร์กิวเมนต์มีชนิดเป็น `number` หรือไม่ก่อน

- `Number.isNaN(value)` คืนค่า `true` ถ้าอาร์กิวเมนต์มีชนิดเป็น `number` และเป็น `NaN` นอกนั้นคืนค่า `false`

    ```js run
    alert( Number.isNaN(NaN) ); // true
    alert( Number.isNaN("str" / 2) ); // true

    // สังเกตความแตกต่าง:
    alert( Number.isNaN("str") ); // false เพราะ "str" เป็นชนิด string ไม่ใช่ number
    alert( isNaN("str") ); // true เพราะ isNaN แปลง "str" เป็นตัวเลขแล้วได้ NaN
    ```

- `Number.isFinite(value)` คืนค่า `true` ถ้าอาร์กิวเมนต์มีชนิดเป็น `number` และไม่ใช่ `NaN/Infinity/-Infinity` นอกนั้นคืนค่า `false`

    ```js run
    alert( Number.isFinite(123) ); // true
    alert( Number.isFinite(Infinity) ); // false
    alert( Number.isFinite(2 / 0) ); // false

    // สังเกตความแตกต่าง:
    alert( Number.isFinite("123") ); // false เพราะ "123" เป็นชนิด string ไม่ใช่ number
    alert( isFinite("123") ); // true เพราะ isFinite แปลง "123" เป็นตัวเลข 123
    ```

ถ้ามองอีกมุม `Number.isNaN` กับ `Number.isFinite` ตรงไปตรงมากว่า `isNaN` และ `isFinite` แต่ในทางปฏิบัติ `isNaN` กับ `isFinite` มักถูกใช้มากกว่า เพราะเขียนสั้นกว่า
````

```smart header="เปรียบเทียบกับ `Object.is`"

มีเมธอดพิเศษในตัว [`Object.is`](mdn:js/Object/is) ที่เปรียบเทียบค่าเหมือน `===` แต่น่าเชื่อถือกว่าในสองกรณีพิเศษ:

1. ทำงานกับ `NaN` ได้: `Object.is(NaN, NaN) === true` ซึ่งดี
2. แยก `0` กับ `-0` ออกจากกัน: `Object.is(0, -0) === false` ทางเทคนิคถูกต้อง เพราะภายในตัวเลขมีบิตเครื่องหมายที่อาจต่างกันแม้บิตอื่นเป็นศูนย์หมด

นอกจากนี้ `Object.is(a, b)` ให้ผลเหมือน `a === b` ทุกประการ

เราพูดถึง `Object.is` ตรงนี้เพราะมักถูกใช้ในสเปคของ JavaScript — เมื่ออัลกอริทึมภายในต้องเปรียบเทียบสองค่าว่าเหมือนกันแบบพอดิบพอดี จะใช้ `Object.is` (เรียกภายในว่า [SameValue](https://tc39.github.io/ecma262/#sec-samevalue))
```


## parseInt และ parseFloat

การแปลงเป็นตัวเลขด้วยเครื่องหมายบวก `+` หรือ `Number()` นั้นเข้มงวด — ถ้าค่าไม่ใช่ตัวเลขพอดี จะล้มเหลว:

```js run
alert( +"100px" ); // NaN
```

ข้อยกเว้นเดียวคือช่องว่างที่อยู่ต้นหรือท้ายสตริง ซึ่งจะถูกละเลย

แต่ในชีวิตจริง เรามักเจอค่าที่มีหน่วย เช่น `"100px"` หรือ `"12pt"` ใน CSS และในหลายประเทศสัญลักษณ์สกุลเงินอยู่หลังจำนวน เช่น `"19€"` แล้วเราก็อยากแยกเฉพาะตัวเลขออกมา

นี่แหละที่ `parseInt` กับ `parseFloat` เข้ามาช่วย

ฟังก์ชันทั้งสอง "อ่าน" ตัวเลขจากสตริงไปเรื่อยๆ จนอ่านไม่ได้ เมื่อเจอ error จะคืนค่าตัวเลขที่อ่านได้แล้ว โดย `parseInt` คืนค่าจำนวนเต็ม ส่วน `parseFloat` คืนค่าทศนิยม:

```js run
alert( parseInt('100px') ); // 100
alert( parseFloat('12.5em') ); // 12.5

alert( parseInt('12.3') ); // 12, คืนเฉพาะส่วนจำนวนเต็ม
alert( parseFloat('12.3.4') ); // 12.3, หยุดอ่านที่จุดทศนิยมที่สอง
```

มีบางสถานการณ์ที่ `parseInt/parseFloat` จะคืนค่า `NaN` เกิดขึ้นเมื่อไม่สามารถอ่านตัวเลขได้เลย:

```js run
alert( parseInt('a123') ); // NaN, หยุดอ่านที่ตัวอักษรแรก
```

````smart header="อาร์กิวเมนต์ที่สองของ parseInt(str, radix)"
ฟังก์ชัน `parseInt()` มีอาร์กิวเมนต์ที่สองเป็นตัวเลือก กำหนดฐานของระบบตัวเลข ดังนั้น `parseInt` สามารถแยกวิเคราะห์สตริงของตัวเลขฐานสิบหก ฐานสอง และอื่นๆ ได้:

```js run
alert( parseInt('0xff', 16) ); // 255
alert( parseInt('ff', 16) ); // 255, ทำงานได้โดยไม่มี 0x ด้วย

alert( parseInt('2n9c', 36) ); // 123456
```
````

## ฟังก์ชันคณิตศาสตร์อื่นๆ

จาวาสคริปต์มีวัตถุ [Math](https://developer.mozilla.org/en/docs/Web/JavaScript/Reference/Global_Objects/Math) ในตัว ซึ่งมีไลบรารีขนาดเล็กของฟังก์ชันคณิตศาสตร์และค่าคงที่

ตัวอย่างบางส่วน:

`Math.random()`
: สร้างตัวเลขสุ่มตั้งแต่ 0 ถึง 1 (ไม่รวม 1)

    ```js run
    alert( Math.random() ); // 0.1234567894322
    alert( Math.random() ); // 0.5435252343232
    alert( Math.random() ); // ... (ตัวเลขสุ่มใดๆ)
    ```

`Math.max(a, b, c...)` / `Math.min(a, b, c...)`
: คืนค่าสูงสุด/ต่ำสุดจากอาร์กิวเมนต์ที่ให้มา

    ```js run
    alert( Math.max(3, 5, -10, 0, 1) ); // 5
    alert( Math.min(1, 2) ); // 1
    ```

`Math.pow(n, power)`
: คืนค่า `n` ยกกำลัง `power`

    ```js run
    alert( Math.pow(2, 10) ); // 2 ยกกำลัง 10 = 1024
    ```

มีฟังก์ชันและค่าคงที่อื่นๆ อีกใน `Math` รวมถึงตรีโกณมิติ ซึ่งคุณสามารถดูได้ในเอกสารสำหรับวัตถุ [Math](https://developer.mozilla.org/en/docs/Web/JavaScript/Reference/Global_Objects/Math)

## สรุป

การเขียนตัวเลขที่มีศูนย์จำนวนมาก:

- เพิ่ม `"e"` ตามด้วยจำนวนศูนย์ต่อท้ายตัวเลข เช่น: `123e6` เท่ากับ `123` ตามด้วยศูนย์ 6 ตัว `123000000`
- ตัวเลขลบหลัง `"e"` ทำให้ตัวเลขถูกหารด้วย 1 ตามด้วยจำนวนศูนย์ที่กำหนด เช่น `123e-6` หมายถึง `0.000123` (`123` หนึ่งในล้าน)

สำหรับระบบตัวเลขต่างๆ:

- เขียนตัวเลขโดยตรงในระบบฐานสิบหก (`0x`), ฐานแปด (`0o`) และฐานสอง (`0b`) ได้
- `parseInt(str, base)` แยกวิเคราะห์สตริง `str` เป็นจำนวนเต็มในระบบตัวเลขฐาน `base`, `2 ≤ base ≤ 36`
- `num.toString(base)` แปลงตัวเลขเป็นสตริงในระบบตัวเลขฐานที่กำหนด

สำหรับการแปลงค่าเช่น `12pt` และ `100px` เป็นตัวเลข:

- ใช้ `parseInt/parseFloat` สำหรับการแปลง "แบบยืดหยุ่น" ซึ่งอ่านตัวเลขจากสตริงแล้วคืนค่าที่อ่านได้ก่อนเกิดข้อผิดพลาด

สำหรับเศษส่วน:

- ปัดเศษโดยใช้ `Math.floor`, `Math.ceil`, `Math.trunc`, `Math.round` หรือ `num.toFixed(precision)`
- พึงระวังการสูญเสียความแม่นยำเมื่อทำงานกับเศษส่วน

ฟังก์ชันคณิตศาสตร์เพิ่มเติม:

- ดู [Math](https://developer.mozilla.org/en/docs/Web/JavaScript/Reference/Global_Objects/Math) เมื่อต้องการใช้ ไลบรารีนี้มีขนาดเล็ก แต่ครอบคลุมความต้องการพื้นฐานทางคณิตศาสตร์ได้
