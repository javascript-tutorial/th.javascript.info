# เมธอดของค่า Primitive

JavaScript ให้เราทำงานกับค่า primitive (เช่น string, number) ราวกับเป็นออบเจ็กต์ โดยมีเมธอดให้เรียกใช้ได้ด้วย เดี๋ยวเราจะมาดูรายละเอียดกัน แต่ก่อนอื่นมาทำความเข้าใจว่ามันทำงานยังไง เพราะแน่นอนว่าค่า primitive ไม่ใช่ออบเจ็กต์ (และเราจะทำให้ชัดเจนยิ่งขึ้น)

มาดูความแตกต่างสำคัญระหว่างค่า primitive กับออบเจ็กต์กัน

ค่า primitive:
- เป็นค่าของชนิดข้อมูล primitive
- มีทั้งหมด 7 ชนิด ได้แก่ `string`, `number`, `bigint`, `boolean`, `symbol`, `null` และ `undefined`

ออบเจ็กต์:
- เก็บข้อมูลได้หลายค่าในรูปแบบพร็อพเพอร์ตี้
- สร้างได้ด้วย `{}` เช่น `{name: "John", age: 30}` นอกจากนี้ JavaScript ยังมีออบเจ็กต์ประเภทอื่นๆ อีก เช่น ฟังก์ชันเองก็ถือเป็นออบเจ็กต์

จุดเด่นอย่างหนึ่งของออบเจ็กต์คือ เราเก็บฟังก์ชันเป็นพร็อพเพอร์ตี้ได้:

```js run
let john = {
  name: "John",
  sayHi: function() {
    alert("Hi buddy!");
  }
};

john.sayHi(); // Hi buddy!
```

ตรงนี้เราสร้างออบเจ็กต์ `john` ที่มีเมธอด `sayHi` ไว้ด้วย

JavaScript มีออบเจ็กต์ในตัวให้ใช้เยอะมาก เช่น ออบเจ็กต์สำหรับวันที่ (Date) ข้อผิดพลาด (Error) องค์ประกอบ HTML เป็นต้น แต่ละตัวก็มีพร็อพเพอร์ตี้และเมธอดเฉพาะของตัวเอง

แต่ความสามารถเหล่านี้ก็มีต้นทุน!

ออบเจ็กต์ "หนัก" กว่าค่า primitive เพราะต้องใช้ทรัพยากรเพิ่มเติมเพื่อรองรับกลไกภายใน

## ค่า Primitive ในฐานะออบเจ็กต์

นี่คือปัญหาที่ผู้สร้าง JavaScript ต้องเผชิญ:

- เรามักอยากทำอะไรหลายๆ อย่างกับค่า primitive เช่น string หรือ number ถ้าเข้าถึงได้ผ่านเมธอดก็จะสะดวกมาก
- แต่ค่า primitive ก็ควรจะเบาและเร็วที่สุดเท่าที่จะเป็นไปได้

วิธีแก้อาจดูแปลกๆ หน่อย แต่เป็นแบบนี้:

1. ค่า primitive ยังคงเป็นค่า primitive — เป็นค่าเดี่ยวตามเดิม
2. ภาษาอนุญาตให้เข้าถึงเมธอดและพร็อพเพอร์ตี้ของ string, number, boolean และ symbol ได้
3. เพื่อให้ทำแบบนั้นได้ จะมีการสร้าง "ออบเจ็กต์ห่อหุ้ม (object wrapper)" พิเศษขึ้นมาชั่วคราว พอใช้เสร็จก็ทำลายทิ้ง

"ออบเจ็กต์ห่อหุ้ม" แต่ละชนิดจะต่างกันไปตามประเภทของค่า primitive ได้แก่ `String`, `Number`, `Boolean`, `Symbol` และ `BigInt` แต่ละตัวก็มีชุดเมธอดเฉพาะของตัวเอง

เช่น เมธอด [str.toUpperCase()](https://developer.mozilla.org/en/docs/Web/JavaScript/Reference/Global_Objects/String/toUpperCase) ที่คืนค่า `str` เป็นตัวพิมพ์ใหญ่ทั้งหมด

ลองดูตัวอย่าง:

```js run
let str = "Hello";

alert( str.toUpperCase() ); // HELLO
```

ง่ายมากใช่ไหม? ทีนี้มาดูว่าเบื้องหลัง `str.toUpperCase()` เกิดอะไรขึ้นบ้าง:

1. string `str` เป็นค่า primitive ดังนั้นตอนที่เข้าถึงพร็อพเพอร์ตี้ จะมีออบเจ็กต์พิเศษถูกสร้างขึ้นมา — ออบเจ็กต์นี้รู้ค่าของ string และมีเมธอดที่มีประโยชน์ต่างๆ เช่น `toUpperCase()`
2. เมธอดทำงานและคืนค่า string ใหม่ (แสดงผ่าน `alert`)
3. ออบเจ็กต์พิเศษถูกทำลายทิ้ง เหลือแค่ค่า primitive `str` ตามเดิม

ค่า primitive จึงเรียกใช้เมธอดได้ แต่ยังคงเบาเหมือนเดิม

JavaScript engine ทำให้กระบวนการนี้เร็วมาก บางครั้งอาจข้ามการสร้างออบเจ็กต์พิเศษไปเลยด้วยซ้ำ แต่ยังคงทำงานตามสเปคเหมือนมีการสร้างออบเจ็กต์ขึ้นมาจริง

ตัวเลขก็มีเมธอดของตัวเองเช่นกัน เช่น [toFixed(n)](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Number/toFixed) ที่ปัดเศษทศนิยมตามจำนวนตำแหน่งที่ระบุ:

```js run
let n = 1.23456;

alert( n.toFixed(2) ); // 1.23
```

เราจะได้เรียนรู้เมธอดเฉพาะของแต่ละชนิดเพิ่มเติมในบท <info:number> และ <info:string>


````warn header="คอนสตรักเตอร์ `String/Number/Boolean` มีไว้ใช้ภายในเท่านั้น"
บางภาษาอย่าง Java อนุญาตให้สร้าง "ออบเจ็กต์ wrapper" สำหรับค่า primitive ได้ เช่น `new Number(1)` หรือ `new Boolean(false)`

ใน JavaScript ก็ทำแบบนั้นได้เหมือนกันเพราะเหตุผลทางประวัติศาสตร์ แต่**ไม่แนะนำอย่างยิ่ง** เพราะจะทำให้เกิดปัญหาหลายอย่าง

เช่น:

```js run
alert( typeof 0 ); // "number"

alert( typeof new Number(0) ); // "object"!
```

เนื่องจากออบเจ็กต์จะมีค่าเป็น truthy เสมอใน `if` ดังนั้น alert ด้านล่างจะแสดงผลออกมา:

```js run
let zero = new Number(0);

if (zero) { // zero เป็น truthy เพราะเป็นออบเจ็กต์
  alert( "zero is truthy!?!" );
}
```

ในทางกลับกัน การใช้ฟังก์ชัน `String/Number/Boolean` แบบไม่มี `new` นั้นโอเคและมีประโยชน์มาก เพราะจะแปลงค่าไปเป็นชนิดที่ต้องการ — ได้ผลลัพธ์เป็นค่า primitive

เช่น แบบนี้ใช้ได้เลย:
```js
let num = Number("123"); // แปลงจาก string เป็น number
```
````

````warn header="null/undefined ไม่มีเมธอด"
ค่า primitive พิเศษอย่าง `null` กับ `undefined` เป็นข้อยกเว้น — ไม่มี "ออบเจ็กต์ wrapper" และไม่มีเมธอดใดๆ เลย พูดได้ว่าเป็นค่า primitive "บริสุทธิ์" ที่สุด

ถ้าพยายามเข้าถึงพร็อพเพอร์ตี้ของค่าเหล่านี้จะเกิด error ทันที:

```js run
alert(null.test); // error
```
````

## สรุป

- ค่า primitive (ยกเว้น `null` กับ `undefined`) มีเมธอดที่มีประโยชน์มากมาย เราจะเรียนรู้เมธอดเหล่านั้นในบทต่อๆ ไป
- เบื้องหลังแล้วเมธอดเหล่านี้ทำงานผ่านออบเจ็กต์ชั่วคราว แต่ JavaScript engine ปรับแต่งมาอย่างดี จึงเรียกใช้ได้โดยไม่ต้องกังวลเรื่องประสิทธิภาพ
