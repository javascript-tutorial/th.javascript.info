# วันที่และเวลา

มาทำความรู้จักกับออบเจ็กต์ built-in ตัวใหม่: [Date](mdn:js/Date) ออบเจ็กต์นี้จัดเก็บวันที่และเวลา และมีเมธอดสำหรับจัดการข้อมูลวันที่/เวลา

ตัวอย่างเช่น เราสามารถใช้มันเพื่อบันทึกเวลาที่สร้างหรือแก้ไขไฟล์ วัดระยะเวลา หรือแค่แสดงวันที่ปัจจุบันก็ได้

## การสร้าง

ในการสร้างออบเจ็กต์ `Date` ใหม่ ให้เรียก `new Date()` พร้อมอาร์กิวเมนต์ในรูปแบบใดรูปแบบหนึ่งดังนี้:

`new Date()`
: ไม่มีอาร์กิวเมนต์ -- สร้างออบเจ็กต์ `Date` สำหรับวันที่และเวลาปัจจุบัน:

    ```js run
    let now = new Date();
    alert( now ); // แสดงวันที่/เวลาปัจจุบัน
    ```

`new Date(milliseconds)`
: สร้างออบเจ็กต์ `Date` โดยกำหนดเวลาจากจำนวนมิลลิวินาที (1/1000 ของวินาที) ที่ผ่านมานับตั้งแต่วันที่ 1 มกราคม 1970 UTC+0

    ```js run
    // 0 หมายถึง 01.01.1970 UTC+0
    let Jan01_1970 = new Date(0);
    alert( Jan01_1970 );

    // เพิ่ม 24 ชั่วโมง ได้ 02.01.1970 UTC+0
    let Jan02_1970 = new Date(24 * 3600 * 1000);
    alert( Jan02_1970 );
    ```

    ตัวเลขจำนวนเต็มที่แสดงจำนวนมิลลิวินาทีที่ผ่านมาตั้งแต่ต้นปี 1970 เรียกว่า *timestamp*

    มันเป็นการแทนค่าวันที่ในรูปแบบตัวเลขที่เบาและกะทัดรัด เราสามารถสร้างวันที่จาก timestamp ได้ตลอดเวลาด้วย `new Date(timestamp)` และแปลงออบเจ็กต์ `Date` เป็น timestamp ด้วยเมธอด `date.getTime()` (ดูรายละเอียดด้านล่าง)

    วันที่ก่อน 01.01.1970 จะมี timestamp เป็นค่าลบ เช่น:
    ```js run
    // 31 ธ.ค. 1969
    let Dec31_1969 = new Date(-24 * 3600 * 1000);
    alert( Dec31_1969 );
    ```

`new Date(datestring)`
: ถ้ามีอาร์กิวเมนต์เพียงตัวเดียวและเป็นสตริง จะถูก parse อัตโนมัติ โดยใช้อัลกอริทึมเดียวกับที่ `Date.parse` ใช้ ซึ่งจะกล่าวถึงในภายหลัง

    ```js run
    let date = new Date("2017-01-26");
    alert(date);
    // ไม่ได้กำหนดเวลา ระบบจึงถือว่าเป็นเที่ยงคืน GMT และ
    // ปรับตามเขตเวลาที่รันโค้ดอยู่
    // ผลลัพธ์อาจเป็น
    // Thu Jan 26 2017 11:00:00 GMT+1100 (Australian Eastern Daylight Time)
    // หรือ
    // Wed Jan 25 2017 16:00:00 GMT-0800 (Pacific Standard Time)
    ```

`new Date(year, month, date, hours, minutes, seconds, ms)`
: สร้างวันที่ด้วยค่าที่กำหนดในเขตเวลาท้องถิ่น โดยอาร์กิวเมนต์สองตัวแรกเท่านั้นที่จำเป็น

    - `year` ควรมี 4 หลัก เพื่อความเข้ากันได้ ยังรับ 2 หลักได้และถือว่าเป็น `19xx` เช่น `98` เทียบเท่ากับ `1998` แต่แนะนำให้ใช้ 4 หลักเสมอ
    - `month` เริ่มนับจาก `0` (ม.ค.) ไปถึง `11` (ธ.ค.)
    - พารามิเตอร์ `date` คือวันที่ในเดือน ถ้าไม่ระบุจะใช้ค่า `1`
    - ถ้าไม่ระบุ `hours/minutes/seconds/ms` จะมีค่าเป็น `0`

    ตัวอย่าง:

    ```js
    new Date(2011, 0, 1, 0, 0, 0, 0); // 1 ม.ค. 2011, 00:00:00
    new Date(2011, 0, 1); // เหมือนกัน ค่า hours เป็นต้นจะเป็น 0 โดยค่าเริ่มต้น
    ```

    ความละเอียดสูงสุดคือ 1 ms (1/1000 วินาที):

    ```js run
    let date = new Date(2011, 0, 1, 2, 3, 4, 567);
    alert( date ); // 1.01.2011, 02:03:04.567
    ```

## การเข้าถึงส่วนประกอบของวันที่

มีเมธอดสำหรับดึงค่าปี เดือน และข้อมูลอื่นๆ จากออบเจ็กต์ `Date`:

[getFullYear()](mdn:js/Date/getFullYear)
: ดึงปี (4 หลัก)

[getMonth()](mdn:js/Date/getMonth)
: ดึงเดือน **ตั้งแต่ 0 ถึง 11**

[getDate()](mdn:js/Date/getDate)
: ดึงวันที่ในเดือน ตั้งแต่ 1 ถึง 31 ชื่อเมธอดนี้ดูแปลกๆ นิดหน่อย

[getHours()](mdn:js/Date/getHours), [getMinutes()](mdn:js/Date/getMinutes), [getSeconds()](mdn:js/Date/getSeconds), [getMilliseconds()](mdn:js/Date/getMilliseconds)
: ดึงค่าส่วนประกอบของเวลาที่สอดคล้องกัน

```warn header="ไม่ใช่ `getYear()` แต่ใช้ `getFullYear()`"
JavaScript engine หลายตัวมีเมธอดที่ไม่ได้มาตรฐานชื่อ `getYear()` เมธอดนี้ถูก deprecated แล้วและบางครั้งคืนค่าปีแค่ 2 หลัก ขอให้หลีกเลี่ยงอย่างเด็ดขาด ให้ใช้ `getFullYear()` แทน
```

นอกจากนี้ยังดึงวันในสัปดาห์ได้:

[getDay()](mdn:js/Date/getDay)
: ดึงวันในสัปดาห์ ตั้งแต่ `0` (อาทิตย์) ถึง `6` (เสาร์) วันแรกเป็นวันอาทิตย์เสมอ บางประเทศอาจไม่ใช่แบบนี้ แต่ก็ไม่สามารถเปลี่ยนแปลงได้

**เมธอดทั้งหมดข้างต้นคืนค่าส่วนประกอบตามเขตเวลาท้องถิ่น**

ยังมีเวอร์ชัน UTC สำหรับแต่ละเมธอด ซึ่งคืนค่าวัน เดือน ปีและอื่นๆ สำหรับเขตเวลา UTC+0: [getUTCFullYear()](mdn:js/Date/getUTCFullYear), [getUTCMonth()](mdn:js/Date/getUTCMonth), [getUTCDay()](mdn:js/Date/getUTCDay) แค่เพิ่ม `"UTC"` ต่อจาก `"get"` ก็พอ

ถ้าเขตเวลาท้องถิ่นของคุณต่างจาก UTC โค้ดด้านล่างจะแสดงชั่วโมงที่ต่างกัน:

```js run
// วันที่ปัจจุบัน
let date = new Date();

// ชั่วโมงในเขตเวลาปัจจุบัน
alert( date.getHours() );

// ชั่วโมงในเขตเวลา UTC+0 (เวลาลอนดอนไม่มีการปรับเวลาฤดูร้อน)
alert( date.getUTCHours() );
```

นอกจากเมธอดที่กล่าวมา ยังมีอีกสองเมธอดพิเศษที่ไม่มีเวอร์ชัน UTC:

[getTime()](mdn:js/Date/getTime)
: คืนค่า timestamp ของวันที่นั้น -- จำนวนมิลลิวินาทีที่ผ่านมาตั้งแต่วันที่ 1 มกราคม 1970 UTC+0

[getTimezoneOffset()](mdn:js/Date/getTimezoneOffset)
: คืนค่าความต่างระหว่าง UTC กับเขตเวลาท้องถิ่น เป็นหน่วยนาที:

    ```js run
    // ถ้าอยู่ในเขตเวลา UTC-1 จะแสดง 60
    // ถ้าอยู่ในเขตเวลา UTC+3 จะแสดง -180
    alert( new Date().getTimezoneOffset() );

    ```

## การกำหนดค่าส่วนประกอบของวันที่

เมธอดต่อไปนี้ใช้สำหรับกำหนดค่าส่วนประกอบของวันที่/เวลา:

- [`setFullYear(year, [month], [date])`](mdn:js/Date/setFullYear)
- [`setMonth(month, [date])`](mdn:js/Date/setMonth)
- [`setDate(date)`](mdn:js/Date/setDate)
- [`setHours(hour, [min], [sec], [ms])`](mdn:js/Date/setHours)
- [`setMinutes(min, [sec], [ms])`](mdn:js/Date/setMinutes)
- [`setSeconds(sec, [ms])`](mdn:js/Date/setSeconds)
- [`setMilliseconds(ms)`](mdn:js/Date/setMilliseconds)
- [`setTime(milliseconds)`](mdn:js/Date/setTime) (กำหนดวันที่ทั้งหมดด้วยมิลลิวินาทีตั้งแต่ 01.01.1970 UTC)

ทุกเมธอดยกเว้น `setTime()` มีเวอร์ชัน UTC เช่น `setUTCHours()`

จะเห็นว่าบางเมธอดสามารถกำหนดหลายค่าพร้อมกันได้ เช่น `setHours` ส่วนค่าที่ไม่ได้ระบุจะไม่ถูกเปลี่ยน

ตัวอย่าง:

```js run
let today = new Date();

today.setHours(0);
alert(today); // ยังเป็นวันนี้อยู่ แต่ชั่วโมงเปลี่ยนเป็น 0

today.setHours(0, 0, 0, 0);
alert(today); // ยังเป็นวันนี้อยู่ แต่ตอนนี้เป็น 00:00:00 พอดี
```

## การแก้ไขค่าอัตโนมัติ

*การแก้ไขค่าอัตโนมัติ (autocorrection)* เป็นฟีเจอร์ที่มีประโยชน์มากของออบเจ็กต์ `Date` เราสามารถกำหนดค่าที่เกินขอบเขตได้ และมันจะปรับค่าให้ถูกต้องเอง

ตัวอย่าง:

```js run
let date = new Date(2013, 0, *!*32*/!*); // 32 ม.ค. 2013 ?!?
alert(date); // ...กลายเป็น 1 ก.พ. 2013!
```

ส่วนประกอบที่เกินขอบเขตจะถูกกระจายออกไปอัตโนมัติ

สมมติว่าต้องการเพิ่ม 2 วันให้กับ "28 ก.พ. 2016" ผลลัพธ์อาจเป็น "2 มี.ค." หรือ "1 มี.ค." ถ้าปีนั้นเป็นปีอธิกสุรทิน เราไม่ต้องคิดเรื่องนี้เลย แค่บวก 2 วัน ออบเจ็กต์ `Date` จัดการให้เอง:

```js run
let date = new Date(2016, 1, 28);
*!*
date.setDate(date.getDate() + 2);
*/!*

alert( date ); // 1 มี.ค. 2016
```

ฟีเจอร์นี้มักใช้เพื่อหาวันที่หลังจากช่วงเวลาที่กำหนด เช่น มาหาวันที่ "70 วินาทีหลังจากนี้":

```js run
let date = new Date();
date.setSeconds(date.getSeconds() + 70);

alert( date ); // แสดงวันที่ที่ถูกต้อง
```

นอกจากนี้ยังกำหนดค่าศูนย์หรือค่าลบได้ด้วย เช่น:

```js run
let date = new Date(2016, 0, 2); // 2 ม.ค. 2016

date.setDate(1); // กำหนดเป็นวันที่ 1 ของเดือน
alert( date );

date.setDate(0); // วันที่น้อยสุดคือ 1 ระบบจึงถือว่าเป็นวันสุดท้ายของเดือนก่อนหน้า
alert( date ); // 31 ธ.ค. 2015
```

## วันที่เป็นตัวเลข และการลบวันที่

เมื่อแปลงออบเจ็กต์ `Date` เป็นตัวเลข จะได้ค่า timestamp เหมือนกับ `date.getTime()`:

```js run
let date = new Date();
alert(+date); // จำนวนมิลลิวินาที เหมือนกับ date.getTime()
```

ผลข้างเคียงที่สำคัญคือ วันที่สามารถนำมาลบกันได้ และได้ผลลัพธ์เป็นความต่างในหน่วย ms

ใช้ประโยชน์ได้ในการวัดเวลา:

```js run
let start = new Date(); // เริ่มจับเวลา

// ทำงานบางอย่าง
for (let i = 0; i < 100000; i++) {
  let doSomething = i * i * i;
}

let end = new Date(); // หยุดจับเวลา

alert( `ลูปใช้เวลา ${end - start} ms` );
```

## Date.now()

ถ้าต้องการแค่วัดเวลา ไม่จำเป็นต้องสร้างออบเจ็กต์ `Date` เลย

มีเมธอดพิเศษ `Date.now()` ที่คืนค่า timestamp ปัจจุบัน

ความหมายเทียบเท่ากับ `new Date().getTime()` แต่ไม่สร้างออบเจ็กต์ `Date` ชั่วคราว จึงเร็วกว่าและไม่กดดันการเก็บขยะ (garbage collection)

มักใช้เพื่อความสะดวกหรือเมื่อประสิทธิภาพสำคัญ เช่นในเกม JavaScript หรือแอปพลิเคชันเฉพาะทาง

วิธีนี้จึงน่าจะดีกว่า:

```js run
*!*
let start = Date.now(); // นับมิลลิวินาทีตั้งแต่ 1 ม.ค. 1970
*/!*

// ทำงานบางอย่าง
for (let i = 0; i < 100000; i++) {
  let doSomething = i * i * i;
}

*!*
let end = Date.now(); // เสร็จแล้ว
*/!*

alert( `ลูปใช้เวลา ${end - start} ms` ); // ลบตัวเลข ไม่ใช่วันที่
```

## Benchmarking

ถ้าต้องการทำ benchmark ที่เชื่อถือได้สำหรับฟังก์ชันที่ใช้ CPU มาก ต้องระมัดระวัง

ลองวัดสองฟังก์ชันที่คำนวณความต่างระหว่างวันที่สองวัน: ฟังก์ชันไหนเร็วกว่า?

การวัดประสิทธิภาพแบบนี้มักเรียกว่า "benchmark"

```js
// มี date1 และ date2 ฟังก์ชันไหนคืนค่าความต่างในหน่วย ms ได้เร็วกว่า?
function diffSubtract(date1, date2) {
  return date2 - date1;
}

// หรือ
function diffGetTime(date1, date2) {
  return date2.getTime() - date1.getTime();
}
```

ทั้งสองทำสิ่งเดียวกันทุกประการ แต่ฟังก์ชันหนึ่งใช้ `date.getTime()` อย่างชัดเจนเพื่อดึงค่าวันที่เป็น ms ส่วนอีกฟังก์ชันอาศัยการแปลงวันที่เป็นตัวเลข ผลลัพธ์เหมือนกันเสมอ

แล้วฟังก์ชันไหนเร็วกว่ากัน?

ความคิดแรกอาจเป็นการรันซ้ำหลายๆ ครั้งติดกันแล้ววัดความต่างของเวลา สำหรับกรณีนี้ฟังก์ชันเรียบง่ายมาก จึงต้องรันอย่างน้อย 100,000 ครั้ง

มาวัดกัน:

```js run
function diffSubtract(date1, date2) {
  return date2 - date1;
}

function diffGetTime(date1, date2) {
  return date2.getTime() - date1.getTime();
}

function bench(f) {
  let date1 = new Date(0);
  let date2 = new Date();

  let start = Date.now();
  for (let i = 0; i < 100000; i++) f(date1, date2);
  return Date.now() - start;
}

alert( 'เวลาของ diffSubtract: ' + bench(diffSubtract) + 'ms' );
alert( 'เวลาของ diffGetTime: ' + bench(diffGetTime) + 'ms' );
```

ว้าว! การใช้ `getTime()` เร็วกว่ามากเลย! เพราะไม่มีการแปลงประเภท engine จึง optimize ได้ง่ายกว่า

โอเค เราได้ข้อมูลมาแล้ว แต่นี่ยังไม่ใช่ benchmark ที่ดี

ลองนึกภาพว่าตอนที่รัน `bench(diffSubtract)` CPU กำลังทำงานอื่นอยู่พร้อมกันและกินทรัพยากรไป พอถึงเวลารัน `bench(diffGetTime)` งานนั้นเสร็จแล้ว

เป็นสถานการณ์จริงที่พบได้บ่อยในระบบปฏิบัติการแบบ multi-process สมัยใหม่

ผลก็คือ benchmark แรกจะได้ทรัพยากร CPU น้อยกว่า benchmark ที่สอง ซึ่งอาจนำไปสู่ผลลัพธ์ที่ผิดพลาดได้

**เพื่อให้ benchmark น่าเชื่อถือกว่า ควรรันชุด benchmark ทั้งหมดหลายๆ รอบ**

ตัวอย่าง:

```js run
function diffSubtract(date1, date2) {
  return date2 - date1;
}

function diffGetTime(date1, date2) {
  return date2.getTime() - date1.getTime();
}

function bench(f) {
  let date1 = new Date(0);
  let date2 = new Date();

  let start = Date.now();
  for (let i = 0; i < 100000; i++) f(date1, date2);
  return Date.now() - start;
}

let time1 = 0;
let time2 = 0;

*!*
// รัน bench(diffSubtract) และ bench(diffGetTime) สลับกัน 10 รอบ
for (let i = 0; i < 10; i++) {
  time1 += bench(diffSubtract);
  time2 += bench(diffGetTime);
}
*/!*

alert( 'เวลารวมของ diffSubtract: ' + time1 );
alert( 'เวลารวมของ diffGetTime: ' + time2 );
```

JavaScript engine สมัยใหม่จะเริ่มใช้การ optimize ขั้นสูงเฉพาะกับ "hot code" ที่ทำงานซ้ำหลายครั้ง (ไม่จำเป็นต้อง optimize สิ่งที่รันไม่บ่อย) ดังนั้นในตัวอย่างข้างต้น การรันครั้งแรกๆ ยังไม่ได้รับการ optimize เราอาจต้องการเพิ่มการ "อุ่นเครื่อง" ก่อน:

```js
// เพิ่มเพื่อ "อุ่นเครื่อง" ก่อนเข้าลูปหลัก
bench(diffSubtract);
bench(diffGetTime);

// เริ่ม benchmark จริง
for (let i = 0; i < 10; i++) {
  time1 += bench(diffSubtract);
  time2 += bench(diffGetTime);
}
```

```warn header="ระวังการทำ microbenchmarking"
JavaScript engine สมัยใหม่ทำการ optimize หลายอย่าง ซึ่งอาจบิดเบือนผลของ "การทดสอบเทียม" เมื่อเทียบกับ "การใช้งานจริง" โดยเฉพาะเมื่อ benchmark สิ่งที่เล็กมาก เช่น วิธีที่ตัวดำเนินการทำงาน หรือฟังก์ชัน built-in ดังนั้นถ้าต้องการเข้าใจประสิทธิภาพอย่างจริงจัง โปรดศึกษาว่า JavaScript engine ทำงานอย่างไร แล้วจะพบว่าแทบไม่จำเป็นต้องใช้ microbenchmark เลย

บทความดีๆ เกี่ยวกับ V8 สามารถหาได้ที่ <https://mrale.ph>

## Date.parse จากสตริง

เมธอด [Date.parse(str)](mdn:js/Date/parse) สามารถอ่านวันที่จากสตริงได้

รูปแบบสตริงควรเป็น: `YYYY-MM-DDTHH:mm:ss.sssZ` โดยที่:

- `YYYY-MM-DD` -- คือวันที่: ปี-เดือน-วัน
- อักขระ `"T"` ใช้เป็นตัวคั่น
- `HH:mm:ss.sss` -- คือเวลา: ชั่วโมง นาที วินาที และมิลลิวินาที
- ส่วน `'Z'` ที่เป็น optional ระบุเขตเวลาในรูปแบบ `+-hh:mm` ถ้าเป็นตัวอักษร `Z` เพียงตัวเดียว หมายถึง UTC+0

รูปแบบย่อก็ได้เช่นกัน เช่น `YYYY-MM-DD` หรือ `YYYY-MM` หรือแม้แต่ `YYYY`

การเรียก `Date.parse(str)` จะ parse สตริงตามรูปแบบที่กำหนดและคืนค่า timestamp (จำนวนมิลลิวินาทีตั้งแต่ 1 ม.ค. 1970 UTC+0) ถ้ารูปแบบไม่ถูกต้อง จะคืนค่า `NaN`

ตัวอย่าง:

```js run
let ms = Date.parse('2012-01-26T13:51:50.417-07:00');

alert(ms); // 1327611110417  (timestamp)
```

เราสามารถสร้างออบเจ็กต์ `new Date` จาก timestamp ได้ทันที:

```js run
let date = new Date( Date.parse('2012-01-26T13:51:50.417-07:00') );

alert(date);
```

## สรุป

- วันที่และเวลาใน JavaScript ใช้ออบเจ็กต์ [Date](mdn:js/Date) เราไม่สามารถสร้าง "แค่วันที่" หรือ "แค่เวลา" ได้: ออบเจ็กต์ `Date` มีทั้งสองเสมอ
- เดือนนับจากศูนย์ (ใช่ มกราคมคือเดือนที่ศูนย์)
- วันในสัปดาห์ใน `getDay()` ก็นับจากศูนย์เช่นกัน (นั่นคือวันอาทิตย์)
- `Date` จะแก้ไขค่าตัวเองอัตโนมัติเมื่อกำหนดค่าที่เกินขอบเขต สะดวกสำหรับการบวก/ลบวัน/เดือน/ชั่วโมง
- วันที่สามารถลบกันได้ ได้ผลต่างในหน่วยมิลลิวินาที เพราะออบเจ็กต์ `Date` กลายเป็น timestamp เมื่อแปลงเป็นตัวเลข
- ใช้ `Date.now()` เพื่อดึง timestamp ปัจจุบันได้อย่างรวดเร็ว

โปรดทราบว่าต่างจากระบบอื่นๆ หลายระบบ timestamp ใน JavaScript มีหน่วยเป็นมิลลิวินาที ไม่ใช่วินาที

บางครั้งเราต้องการวัดเวลาที่แม่นยำกว่านี้ JavaScript เองไม่มีวิธีวัดเวลาในระดับไมโครวินาที (1 ในล้านของวินาที) แต่สภาพแวดล้อมส่วนใหญ่มีให้ เช่น เบราว์เซอร์มี [performance.now()](mdn:api/Performance/now) ที่คืนค่าจำนวนมิลลิวินาทีตั้งแต่เริ่มโหลดหน้าเว็บพร้อมความแม่นยำระดับไมโครวินาที (3 ตำแหน่งหลังจุดทศนิยม):

```js run
alert(`โหลดเริ่มมาแล้ว ${performance.now()}ms`);
// บางอย่างเช่น: "โหลดเริ่มมาแล้ว 34731.26000000001ms"
// .26 คือไมโครวินาที (260 ไมโครวินาที)
// ตัวเลขหลังจุดทศนิยมมากกว่า 3 ตำแหน่งคือความผิดพลาดของความแม่นยำ ถูกต้องแค่ 3 ตำแหน่งแรก
```

Node.js มีโมดูล `microtime` และวิธีอื่นๆ ในทางเทคนิค แทบทุกอุปกรณ์และสภาพแวดล้อมสามารถให้ความแม่นยำสูงกว่านี้ได้ เพียงแต่ยังไม่อยู่ใน `Date`
