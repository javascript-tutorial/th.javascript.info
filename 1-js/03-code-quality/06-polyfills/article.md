# Polyfills และ Transpilers

ภาษา JavaScript วิวัฒนาการอยู่ตลอดเวลา มีข้อเสนอใหม่ๆ เข้ามาเป็นประจำ ซึ่งจะถูกวิเคราะห์และพิจารณาก่อนจะเข้าไปอยู่ในรายการที่ <https://tc39.github.io/ecma262/> แล้วค่อยๆ ผ่านเข้าสู่[ข้อกำหนด](https://www.ecma-international.org/publications-and-standards/standards/ecma-262/)

ทีมงานเบื้องหลังเอนจิ้น JavaScript แต่ละค่ายก็มีแนวคิดของตัวเองว่าจะเริ่มใช้อะไรก่อน บางทีอาจเลือกใช้ข้อเสนอที่ยังอยู่ในร่าง แต่เลื่อนสิ่งที่อยู่ในข้อกำหนดแล้วออกไป — เพราะน่าสนใจน้อยกว่าหรือทำยากกว่า

ดังนั้นจึงเป็นเรื่องปกติที่เอนจิ้นจะรองรับแค่บางส่วนของมาตรฐานเท่านั้น

เว็บเพจที่ดีสำหรับดูสถานะการรองรับฟีเจอร์คือ <https://compat-table.github.io/compat-table/es6/> (ข้อมูลเยอะมาก เรายังมีอีกหลายอย่างที่ต้องศึกษา)

ในฐานะโปรแกรมเมอร์ เราอยากใช้ฟีเจอร์ล่าสุด ยิ่งมีของดีๆ มากเท่าไหร่ก็ยิ่งดี!

แต่แล้วจะทำอย่างไรให้โค้ดสมัยใหม่ของเราทำงานได้บนเอนจิ้นเก่าๆ ที่ยังไม่เข้าใจฟีเจอร์ใหม่ล่ะ?

มีเครื่องมือสองอย่างที่ช่วยได้:

1. Transpilers
2. Polyfills

ในบทนี้เราจะมาทำความเข้าใจเบื้องต้นว่าเครื่องมือเหล่านี้ทำงานอย่างไร และมีบทบาทอะไรในการพัฒนาเว็บ

## Transpilers

[Transpiler](https://en.wikipedia.org/wiki/Source-to-source_compiler) เป็นซอฟต์แวร์พิเศษที่แปลซอร์สโค้ดชุดหนึ่งเป็นซอร์สโค้ดอีกชุดหนึ่ง โดยจะอ่านและทำความเข้าใจโค้ดสมัยใหม่ แล้วเขียนใหม่ด้วยไวยากรณ์แบบเก่า เพื่อให้ทำงานได้แม้บนเอนจิ้นที่ล้าสมัย

เช่น JavaScript ก่อนปี 2020 ไม่มี "nullish coalescing operator" `??` ถ้าผู้ใช้เปิดเบราว์เซอร์รุ่นเก่า โค้ดอย่าง `height = height ?? 100` ก็อาจพังได้เลย

Transpiler จะวิเคราะห์โค้ดของเราและเขียน `height ?? 100` ใหม่เป็น `(height !== undefined && height !== null) ? height : 100`

```js
// ก่อนรัน transpiler
height = height ?? 100;

// หลังรัน transpiler  
height = (height !== undefined && height !== null) ? height : 100;
```

แบบนี้โค้ดก็ใช้ได้กับเอนจิ้น JavaScript รุ่นเก่าแล้ว

โดยปกตินักพัฒนาจะรัน transpiler บนเครื่องตัวเอง แล้วค่อยนำโค้ดที่ transpile แล้วไปขึ้นเซิร์ฟเวอร์

พูดถึงชื่อ [Babel](https://babeljs.io) ถือเป็น transpiler ที่ได้รับความนิยมมากที่สุดตัวหนึ่ง

ระบบ build โปรเจ็กต์สมัยใหม่อย่าง [webpack](https://webpack.js.org/) สามารถรัน transpiler ให้อัตโนมัติทุกครั้งที่โค้ดเปลี่ยน ทำให้รวมเข้ากับขั้นตอนการพัฒนาได้ง่ายมาก

## Polyfills

ฟีเจอร์ใหม่ของภาษาไม่ได้มีแค่ไวยากรณ์และตัวดำเนินการเท่านั้น แต่ยังรวมถึงฟังก์ชันในตัวด้วย

เช่น `Math.trunc(n)` เป็นฟังก์ชันที่ "ตัด" ส่วนทศนิยมออก — `Math.trunc(1.23)` จะคืนค่า `1`

ในเอนจิ้น JavaScript บางตัวที่ล้าสมัยมากอาจไม่มี `Math.trunc` ทำให้โค้ดทำงานไม่ได้

เนื่องจากเรากำลังพูดถึงฟังก์ชันใหม่ ไม่ใช่การเปลี่ยนแปลงไวยากรณ์ จึงไม่ต้อง transpile อะไร แค่ประกาศฟังก์ชันที่ขาดหายไปก็พอ

สคริปต์ที่อัปเดตหรือเพิ่มฟังก์ชันใหม่แบบนี้เรียกว่า "polyfill" — ทำหน้าที่ "เติมเต็ม" ช่องว่างโดยเพิ่มส่วนที่ขาดหายไปให้

polyfill สำหรับ `Math.trunc` ก็จะเป็นสคริปต์ที่เขียน implement ฟังก์ชันนี้ขึ้นมา แบบนี้:

```js
if (!Math.trunc) { // ถ้าไม่มีฟังก์ชันดังกล่าว
  // ก็เขียนขึ้นมาเอง
  Math.trunc = function(number) {
    // Math.ceil และ Math.floor มีอยู่แม้ในเอนจิ้น JavaScript โบราณ
    // จะมีการอธิบายเพิ่มเติมในบทเรียนถัดไป
    return number < 0 ? Math.ceil(number) : Math.floor(number);
  };
}
```

JavaScript เป็นภาษาที่ยืดหยุ่นมาก สคริปต์สามารถเพิ่มหรือแก้ไขฟังก์ชันใดก็ได้ แม้แต่ฟังก์ชันในตัว

ไลบรารี polyfill ที่น่าสนใจ เช่น:
- [core-js](https://github.com/zloirock/core-js) ที่รองรับฟีเจอร์จำนวนมาก และเลือกใช้เฉพาะที่ต้องการได้
- บริการ [polyfill.io](https://polyfill.io/) ที่สร้างสคริปต์พร้อม polyfill ให้ตามฟีเจอร์และเบราว์เซอร์ของผู้ใช้

## สรุป

บทนี้ต้องการชวนให้ลองศึกษาฟีเจอร์ภาษาที่ทันสมัย รวมถึงฟีเจอร์ที่ยังอยู่ระหว่างพัฒนา แม้ว่าเอนจิ้น JavaScript จะยังรองรับได้ไม่ครบก็ตาม

อย่าลืมใช้ transpiler (ถ้าใช้ไวยากรณ์หรือตัวดำเนินการใหม่) และ polyfill (เพื่อเพิ่มฟังก์ชันที่อาจขาดหายไป) เครื่องมือเหล่านี้จะช่วยให้มั่นใจได้ว่าโค้ดจะทำงานได้

เช่น เมื่อคุ้นเคยกับ JavaScript มากขึ้นแล้ว ก็ลองตั้งค่าระบบ build ด้วย [webpack](https://webpack.js.org/) ร่วมกับปลั๊กอิน [babel-loader](https://github.com/babel/babel-loader) ได้

แหล่งข้อมูลดีๆ ที่แสดงสถานะการรองรับฟีเจอร์ต่างๆ ได้แก่:
- <https://compat-table.github.io/compat-table/es6/> — สำหรับ JavaScript แท้ๆ
- <https://caniuse.com/> — สำหรับฟีเจอร์ที่เกี่ยวข้องกับเบราว์เซอร์

ป.ล. Google Chrome มักจะรองรับฟีเจอร์ภาษาใหม่ๆ ได้เร็วที่สุด ถ้าเดโมในบทเรียนไม่ทำงาน ลองใช้ Chrome ดู แต่โดยทั่วไปแล้วเดโมส่วนใหญ่ทำงานได้กับเบราว์เซอร์สมัยใหม่ทุกตัว
